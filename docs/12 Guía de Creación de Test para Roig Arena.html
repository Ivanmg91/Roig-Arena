Esta guía te ayudará a entender qué son los tests, cómo funcionan y cómo escribirlos en Laravel.

¿Qué son los tests?

Los tests son código que verifica que tu código funciona correctamente. Son como un inspector de calidad que revisa automáticamente que todo funcione como esperas.

Analogía simple

Imagina que construyes un coche:

  • Código de producción = El coche
  • Tests = Las pruebas de seguridad y calidad
  • Test que pasa = ✅ El coche funciona
  • Test que falla = ❌ Hay un problema que arreglar

¿Por qué escribir tests?

Sin tests (❌ Problemático)

// Haces un cambio en el código
public function calcularPrecio($base, $descuento)
{
    return $base - $descuento; // ¿Funciona? ¿Seguro?
}

// Tienes que probar manualmente:
// 1. Abrir el navegador
// 2. Hacer login
// 3. Ir a la página
// 4. Probar cada caso
// 5. Repetir cada vez que cambias algo

Con tests (✅ Automatizado)

public function test_calcula_precio_correctamente()
{
    $precio = calcularPrecio(100, 20);
    $this->assertEquals(80, $precio);
}

// Ejecutas: php artisan test
// En 2 segundos sabes si funciona ✅

Ventajas:

  • ✅ Detectas errores inmediatamente
  • ✅ Refactorizas con confianza
  • ✅ Documentación viva del código
  • ✅ Menos bugs en producción
  • ✅ Ahorras tiempo a largo plazo

Tipos de tests en Laravel

1. Feature Tests (Integración)

Prueban funcionalidades completas desde el punto de vista del usuario.

// tests/Feature/AuthTest.php
public function test_usuario_puede_registrarse()
{
    $response = $this->postJson('/api/register', [
        'nombre' => 'Juan',
        'email' => 'juan@example.com',
        'password' => 'password123',
    ]);

    $response->assertStatus(201);
    $this->assertDatabaseHas('users', ['email' => 'juan@example.com']);
}

Características:

  • Prueban el flujo completo (ruta → controlador → modelo → BD)
  • Simulan peticiones HTTP reales
  • Verifican respuestas y base de datos
  • Más lentos pero más completos

2. Unit Tests (Unitarios)

Prueban piezas individuales de código de forma aislada.

// tests/Unit/ReservaServiceTest.php
public function test_puede_reservar_asiento()
{
    $service = new ReservaService();
    $reserva = $service->reservarAsiento(1, 5, 3);
    
    $this->assertNotNull($reserva);
    $this->assertEquals('bloqueado', $reserva->estado);
}

Características:

  • Prueban métodos específicos
  • No hacen peticiones HTTP
  • Más rápidos
  • Más enfocados

Estructura de un test

<?php

namespace Tests\Feature;

use Tests\TestCase;
use Illuminate\Foundation\Testing\RefreshDatabase;

class MiTest extends TestCase
{
    use RefreshDatabase; // Limpia la BD después de cada test
    
    public function test_descripcion_de_lo_que_prueba()
    {
        // 1. ARRANGE - Preparar datos
        $user = User::factory()->create();
        
        // 2. ACT - Ejecutar acción
        $response = $this->actingAs($user)->getJson('/api/eventos');
        
        // 3. ASSERT - Verificar resultado
        $response->assertStatus(200);
    }
}

Patrón AAA (Arrange, Act, Assert)

  1. Arrange - Preparar el escenario
  2. Act - Ejecutar la acción a probar
  3. Assert - Verificar el resultado

Assertions (Verificaciones)

Assertions de respuesta HTTP

// Código de estado
$response->assertStatus(200);        // OK
$response->assertStatus(201);        // Created
$response->assertStatus(404);        // Not Found
$response->assertStatus(422);        // Validation Error

// Estructura JSON
$response->assertJson(['nombre' => 'Juan']);
$response->assertJsonStructure(['data' => ['id', 'nombre']]);
$response->assertJsonCount(3, 'data');

// Redirecciones
$response->assertRedirect('/home');

Assertions de base de datos

// Verificar que existe
$this->assertDatabaseHas('users', ['email' => 'juan@example.com']);

// Verificar que NO existe
$this->assertDatabaseMissing('users', ['email' => 'noexiste@example.com']);

// Verificar soft delete
$this->assertSoftDeleted('eventos', ['id' => 1]);

Assertions generales

// Igualdad
$this->assertEquals(10, $resultado);
$this->assertNotEquals(5, $resultado);

// Verdadero/Falso
$this->assertTrue($condicion);
$this->assertFalse($condicion);

// Null
$this->assertNull($valor);
$this->assertNotNull($valor);

// Arrays
$this->assertCount(3, $array);
$this->assertContains('valor', $array);

// Strings
$this->assertStringContains('texto', $cadena);
$this->assertStringStartsWith('QR-', $codigo);

// Tipos
$this->assertIsInt($numero);
$this->assertIsString($texto);
$this->assertIsArray($datos);

// Comparaciones
$this->assertGreaterThan(5, $numero);
$this->assertLessThan(10, $numero);

RefreshDatabase

El trait RefreshDatabase limpia la base de datos entre tests.

use Illuminate\Foundation\Testing\RefreshDatabase;

class MiTest extends TestCase
{
    use RefreshDatabase;
    
    public function test_ejemplo()
    {
        // La BD está limpia al inicio
        User::create(['email' => 'test@example.com']);
        
        // Al terminar, se revierte todo
    }
}

¿Qué hace?

  • Ejecuta las migraciones antes del primer test
  • Usa transacciones para revertir cambios
  • Cada test empieza con BD limpia

Autenticación en tests

Con Sanctum (API)

public function test_usuario_autenticado()
{
    $user = User::factory()->create();
    $token = $user->createToken('test')->plainTextToken;
    
    $response = $this->withToken($token)->getJson('/api/user');
    
    $response->assertStatus(200);
}

Con actingAs()

public function test_usuario_autenticado()
{
    $user = User::factory()->create();
    
    $response = $this->actingAs($user)->getJson('/api/user');
    
    $response->assertStatus(200);
}

Peticiones HTTP en tests

GET

$response = $this->getJson('/api/eventos');
$response = $this->get('/eventos'); // Sin JSON

POST

$response = $this->postJson('/api/eventos', [
    'nombre' => 'Evento',
    'fecha' => '2026-12-31',
]);

PUT

$response = $this->putJson('/api/eventos/1', [
    'nombre' => 'Evento Actualizado',
]);

DELETE

$response = $this->deleteJson('/api/eventos/1');

Con headers

$response = $this->withHeaders([
    'Authorization' => 'Bearer ' . $token,
    'Accept' => 'application/json',
])->getJson('/api/eventos');

Excepciones en tests

Esperar una excepción

public function test_lanza_excepcion()
{
    $this->expectException(\Exception::class);
    $this->expectExceptionMessage('No disponible');
    
    $service->reservarAsiento(1, 5, 3); // Debe lanzar excepción
}

Mocking (Simulación)

Simular dependencias para aislar el código a probar.

Mock de un servicio

public function test_con_mock()
{
    $mock = $this->mock(EmailService::class);
    $mock->shouldReceive('enviar')
         ->once()
         ->with('test@example.com')
         ->andReturn(true);
    
    // Tu código que usa EmailService
}

Mock de facades

use Illuminate\Support\Facades\Mail;

public function test_envia_email()
{
    Mail::fake();
    
    // Código que envía email
    
    Mail::assertSent(WelcomeEmail::class);
}

Comandos Artisan

# Ejecutar todos los tests
sail artisan test

# Solo Feature tests
sail artisan test --testsuite=Feature

# Solo Unit tests
sail artisan test --testsuite=Unit

# Un archivo específico
sail artisan test tests/Feature/AuthTest.php

# Un test específico
sail artisan test --filter=test_usuario_puede_registrarse

# Detener en el primer fallo
sail artisan test --stop-on-failure

# Con más detalle
sail artisan test --verbose

# En paralelo (más rápido)
sail artisan test --parallel

# Con cobertura de código
sail artisan test --coverage

Crear tests

# Feature test
sail artisan make:test EventoTest

# Unit test
sail artisan make:test ReservaServiceTest --unit

✅ Hacer

TDD (Test-Driven Development)

Escribir tests ANTES del código:

1. RED - Escribir test que falla
2. GREEN - Escribir código mínimo para que pase
3. REFACTOR - Mejorar el código
4. Repetir

Ejemplo:

// 1. RED - Test que falla
public function test_calcula_descuento()
{
    $precio = calcularDescuento(100, 20);
    $this->assertEquals(80, $precio);
}

// 2. GREEN - Código mínimo
function calcularDescuento($base, $descuento)
{
    return $base - $descuento;
}

// 3. REFACTOR - Mejorar
function calcularDescuento($base, $descuento)
{
    if ($descuento > $base) {
        throw new \Exception('Descuento inválido');
    }
    return $base - $descuento;
}

Errores comunes

Error: “Database not found”

Solución:

sail mysql -e "CREATE DATABASE IF NOT EXISTS testing;"

Error: “Class not found”

Solución:

sail composer dump-autoload

Error: Tests muy lentos

Solución: Usar SQLite en memoria

<env name="DB_CONNECTION" value="sqlite"/>
<env name="DB_DATABASE" value=":memory:"/>
