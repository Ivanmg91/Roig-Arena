Guía de Instalación de Docker en Debian

Docker es una plataforma que permite crear, empaquetar y ejecutar aplicaciones dentro de contenedores. Un contenedor incluye la aplicación y todo lo que necesita para funcionar (bibliotecas, dependencias, configuración), de modo que se ejecuta igual en cualquier ordenador o servidor.

En pocas palabras:
“Con Docker, si funciona en mi equipo, funciona en cualquier sitio.”

Esta guía te llevará paso a paso por la instalación de Docker Engine en Debian. Está basada en el script instalar_docker.sh pero explicada de forma clara para que entiendas qué hace cada comando.

¿Por qué estos pasos?

Docker no viene en los repositorios oficiales de Debian con la última versión, así que necesitamos agregar el repositorio oficial de Docker. Además, Debian Trixie (testing) a veces no tiene soporte directo, por lo que usaremos los paquetes de Bookworm (stable) que son totalmente compatibles.

Paso 1: Limpiar instalaciones previas

Si ya intentaste instalar Docker antes y algo salió mal, este comando elimina configuraciones antiguas que podrían causar conflictos:

sudo rm -f /etc/apt/sources.list.d/docker.list

Paso 2: Actualizar el sistema

Siempre es buena idea tener los índices de paquetes actualizados antes de instalar algo nuevo:

sudo apt-get update

Paso 3: Instalar dependencias necesarias

Docker necesita estas herramientas para funcionar correctamente:

  • ca-certificates: Para verificar certificados SSL/TLS
  • curl: Para descargar archivos desde internet
  • gnupg: Para verificar firmas digitales de los paquetes
sudo apt-get install -y ca-certificates curl gnupg

Paso 4: Configurar la clave GPG de Docker

Las claves GPG garantizan que los paquetes que descargas realmente vienen de Docker y no han sido modificados. Aquí creamos un directorio seguro y descargamos la clave oficial:

sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

¿Qué hace cada parte?

  • install -m 0755 -d: Crea el directorio con permisos específicos
  • curl -fsSL: Descarga la clave de forma silenciosa
  • gpg --dearmor: Convierte la clave a formato binario
  • chmod a+r: Da permisos de lectura a todos los usuarios

Paso 5: Agregar el repositorio de Docker

Aquí le decimos a Debian dónde encontrar los paquetes de Docker. Usamos Bookworm porque es la versión estable y siempre tiene soporte completo:

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
  bookworm stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

Desglose:

  • $(dpkg --print-architecture): Detecta automáticamente tu arquitectura (amd64, arm64, etc.)
  • signed-by=...: Indica qué clave GPG usar para verificar los paquetes
  • bookworm stable: Usa los paquetes estables de Debian 12

Paso 6: Instalar Docker

Ahora sí, actualizamos los índices con el nuevo repositorio e instalamos Docker completo:

sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

¿Qué estamos instalando?

  • docker-ce: Docker Community Edition (el motor principal)
  • docker-ce-cli: La interfaz de línea de comandos
  • containerd.io: El runtime de contenedores
  • docker-buildx-plugin: Herramienta avanzada para construir imágenes
  • docker-compose-plugin: Para orquestar múltiples contenedores

Paso 7: Verificar la instalación

Comprobamos que Docker se instaló correctamente y vemos la versión:

if command -v docker &> /dev/null
then
    echo "¡Docker instalado con éxito!"
    docker --version
else
    echo "Hubo un error en la instalación."
fi

Configuración post-instalación

Después de instalar Docker, necesitas hacer algunos ajustes para usarlo cómodamente:

Iniciar el servicio Docker

sudo systemctl start docker
sudo systemctl enable docker
  • start: Inicia Docker ahora mismo
  • enable: Hace que Docker se inicie automáticamente al encender el PC

Agregar tu usuario al grupo docker

Por defecto, solo root puede usar Docker. Para evitar escribir sudo cada vez:

sudo usermod -aG docker $USER

⚠️ IMPORTANTE: Después de este comando, debes cerrar sesión y volver a entrar (o reiniciar) para que los cambios surtan efecto.

Probar que todo funciona

docker run hello-world

Si ves un mensaje de bienvenida, ¡todo está listo!

Solución de problemas comunes

Error: “Cannot connect to the Docker daemon”

Verifica que el servicio esté corriendo:

sudo systemctl status docker

Si está inactivo, inícialo con:

sudo systemctl start docker

Error: “Permission denied”

Asegúrate de haber agregado tu usuario al grupo docker y de haber cerrado sesión después de hacerlo.

Problemas con iptables en Debian Trixie

Debian Trixie usa iptables-nft por defecto, lo que puede causar conflictos. Instala el paquete de compatibilidad:

instalar_docker.sh

sudo apt-get install -y iptables

#!/bin/bash

# 1. Limpiar rastro del intento anterior
sudo rm -f /etc/apt/sources.list.d/docker.list

echo "Actualizando índices de Debian..."
sudo apt-get update

# 2. Instalar dependencias
sudo apt-get install -y ca-certificates curl gnupg

# 3. Llave GPG oficial de Docker
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 4. Configurar repositorio (Usando 'bookworm' como base estable si trixie falla)
# Docker a veces no tiene carpeta 'trixie' todavía, así que usamos la de Debian estable
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
  bookworm stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 5. Instalar
echo "Instalando Docker Engine..."
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 6. Verificación real
if command -v docker &> /dev/null
then
    echo "¡Docker instalado con éxito!"
    docker --version
else
    echo "Hubo un error en la instalación."
fi

¿Y ahora qué?

Con Docker instalado, ya puedes usar Laravel Sail, Docker Compose, y cualquier contenedor que necesites

Guía de Instalación de Laravel Sail

Laravel Sail es el entorno de desarrollo oficial de Laravel basado en Docker. Te permite tener un entorno completo (PHP, MySQL, Redis, etc.) sin instalar nada directamente en tu sistema, solo necesitas Docker.

Requisitos previos

Antes de empezar, asegúrate de tener:

  • Docker instalado y funcionando (puedes verificar con docker --version)
  • Tu usuario agregado al grupo docker (para no usar sudo)
  • Conexión a internet

¿Qué es Laravel Sail?

Sail es una interfaz de línea de comandos ligera para interactuar con la configuración Docker predeterminada de Laravel. En lugar de configurar manualmente PHP, MySQL, Redis y otros servicios, Sail lo hace todo por ti con un solo comando.

Instalación de un nuevo proyecto Laravel con Sail

Laravel ofrece un instalador automático que descarga todo lo necesario y configura Sail en un solo paso.

El comando mágico

curl -s "https://laravel.build/nombre-proyecto" | bash

¿Qué hace este comando?

  • curl -s: Descarga el script de instalación de forma silenciosa
  • https://laravel.build/nombre-proyecto: El instalador oficial de Laravel
  • | bash: Ejecuta el script descargado

Importante: Reemplaza nombre-proyecto con el nombre que quieras para tu aplicación (sin espacios, usa guiones si es necesario).

Ejemplo: Crear proyecto “arena” con MySQL

curl -s "https://laravel.build/arena?with=mysql" | bash

Este comando:

  1. Crea una carpeta llamada arena
  2. Descarga Laravel dentro de ella
  3. Configura Sail con soporte para MySQL
  4. Instala todas las dependencias de Composer

Servicios disponibles

Puedes agregar más servicios usando el parámetro ?with=:

# Solo MySQL
curl -s "https://laravel.build/mi-app?with=mysql" | bash

# MySQL + Redis
curl -s "https://laravel.build/mi-app?with=mysql,redis" | bash

# MySQL + Redis + Mailpit + Meilisearch
curl -s "https://laravel.build/mi-app?with=mysql,redis,mailpit,meilisearch" | bash

Servicios disponibles:

  • mysql - Base de datos MySQL
  • pgsql - Base de datos PostgreSQL
  • mariadb - Base de datos MariaDB
  • redis - Cache y colas
  • memcached - Sistema de caché
  • meilisearch - Motor de búsqueda
  • typesense - Motor de búsqueda alternativo
  • minio - Almacenamiento compatible con S3
  • mailpit - Servidor de correo para desarrollo
  • selenium - Testing de navegador

Tiempo de instalación

La primera vez puede tardar varios minutos porque:

  • Descarga Laravel y todas sus dependencias
  • Descarga las imágenes Docker necesarias (PHP, MySQL, etc.)
  • Configura todo el entorno

Ten paciencia, es normal que tarde entre 5-10 minutos dependiendo de tu conexión.

Después de la instalación

Una vez que termine, entra a la carpeta de tu proyecto:

cd arena

Iniciar Sail

Para levantar todos los contenedores y servicios:

./vendor/bin/sail up

O en modo background (sin ocupar la terminal):

./vendor/bin/sail up -d

¿Qué pasa cuando ejecutas esto?

  • Se inician los contenedores de Docker
  • PHP, MySQL y otros servicios quedan corriendo
  • Tu aplicación estará disponible en http://localhost

Detener Sail

Para apagar todos los servicios:

./vendor/bin/sail down

Crear un alias (recomendado)

Escribir ./vendor/bin/sail cada vez es tedioso. Crea un alias en tu shell:

Para Bash:

echo "alias sail='./vendor/bin/sail'" >> ~/.bashrc
source ~/.bashrc

Para Zsh:

echo "alias sail='./vendor/bin/sail'" >> ~/.zshrc
source ~/.zshrc

Ahora puedes usar simplemente:

sail up
sail down
sail artisan migrate

Comandos útiles de Sail

Ejecutar comandos Artisan

sail artisan migrate
sail artisan make:model Post
sail artisan tinker

Ejecutar comandos Composer

sail composer require laravel/sanctum
sail composer update

Ejecutar comandos NPM

sail npm install
sail npm run dev
sail npm run build

Ejecutar tests

sail test
sail test --filter NombreDelTest

Acceder a la base de datos

sail mysql

O si usas PostgreSQL:

sail psql

Ver logs en tiempo real

sail logs
sail logs -f  # Seguir los logs en tiempo real

Ejecutar comandos dentro del contenedor

sail shell

Esto te da acceso a una terminal bash dentro del contenedor de PHP.

Reiniciar servicios

sail restart

Estructura del proyecto con Sail

Después de la instalación, tu proyecto tendrá:

arena/
├── docker-compose.yml    # Configuración de Docker
├── .env                  # Variables de entorno
├── vendor/
│   └── bin/
│       └── sail          # El comando sail
├── app/                  # Tu código Laravel
├── database/
└── ...

Configuración de la base de datos

Sail configura automáticamente tu archivo .env con las credenciales correctas:

DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=arena
DB_USERNAME=sail
DB_PASSWORD=password

Nota: El host es mysql (nombre del servicio Docker), no localhost ni 127.0.0.1.

Acceder a tu aplicación

Una vez que Sail esté corriendo:

  • Aplicación web: http://localhost
  • Mailpit (si lo instalaste): http://localhost:8025
  • MySQL: localhost:3306 (desde tu máquina host)

Solución de problemas comunes

Error: “Port 80 is already in use”

Otro servicio está usando el puerto 80. Puedes:

  1. Detener el servicio que lo usa (Apache, Nginx, etc.)
  2. O cambiar el puerto en docker-compose.yml:
ports:
    - '8000:80'

Ahora tu app estará en http://localhost:8000

Error: “Cannot connect to Docker daemon”

Docker no está corriendo. Inícialo:

sudo systemctl start docker

O si usas Docker Desktop en Windows/Mac, asegúrate de que esté abierto.

Error: “Permission denied” al ejecutar Sail

Tu usuario no está en el grupo docker:

sudo usermod -aG docker $USER

Luego cierra sesión y vuelve a entrar.

Los cambios en el código no se reflejan

Si estás usando WSL2 con Docker Desktop, asegúrate de que tu proyecto esté en el sistema de archivos de Linux (/home/usuario/...) y no en Windows (/mnt/c/...). Los archivos en /mnt/c/ son mucho más lentos.

Limpiar todo y empezar de cero

Si algo se rompió y quieres resetear:

sail down -v  # Detiene y elimina volúmenes (¡borra la base de datos!)
sail build --no-cache  # Reconstruye las imágenes
sail up

Diferencias entre entornos

En Debian/Linux nativo

Todo funciona directamente. Solo asegúrate de que Docker esté instalado y corriendo.

En WSL2 (Windows)

Si tienes Docker Desktop instalado en Windows:

  • No necesitas instalar Docker dentro de WSL2
  • Docker Desktop se integra automáticamente con WSL2
  • Guarda tus proyectos en /home/usuario/ (no en /mnt/c/)
  • El rendimiento será mucho mejor

En macOS

Similar a Windows, usa Docker Desktop y todo funcionará sin problemas.

¡Todo sin preocuparte por configurar PHP, MySQL o cualquier otra dependencia!

Guía para Añadir phpMyAdmin a Laravel Sail

phpMyAdmin es una herramienta web que te permite administrar tu base de datos MySQL de forma visual, sin necesidad de usar comandos SQL en la terminal. Es perfecta para ver tablas, ejecutar consultas, importar/exportar datos y más.

¿Por qué añadir phpMyAdmin?

Aunque puedes acceder a MySQL desde la terminal con sail mysql, phpMyAdmin ofrece:

  • Interfaz gráfica intuitiva
  • Visualización fácil de tablas y datos
  • Editor SQL con autocompletado
  • Importación/exportación de bases de datos
  • Gestión de usuarios y permisos

Requisitos previos

  • Tener Laravel Sail instalado y funcionando
  • Tener MySQL configurado en tu proyecto (viene por defecto si instalaste con ?with=mysql)

Paso 1: Editar el archivo compose.yaml

Laravel Sail usa Docker Compose para gestionar los servicios. Necesitas añadir phpMyAdmin como un nuevo servicio.

Abre el archivo compose.yaml en la raíz de tu proyecto:

cd arena  # o el nombre de tu proyecto
nano compose.yaml  # o usa tu editor favorito

Paso 2: Añadir el servicio de phpMyAdmin

Busca la sección del servicio mysql (debería estar después del servicio laravel.test). Justo después de la sección de MySQL, añade esto:

    phpmyadmin:
        image: 'phpmyadmin:latest'
        ports:
            - '${FORWARD_PHPMYADMIN_PORT:-8080}:80'
        environment:
            PMA_HOST: mysql
            PMA_USER: '${DB_USERNAME}'
            PMA_PASSWORD: '${DB_PASSWORD}'
            UPLOAD_LIMIT: 300M
        networks:
            - sail
        depends_on:
            - mysql

Importante: Respeta la indentación. En YAML, los espacios son cruciales.

¿Qué hace cada parte?

  • image: 'phpmyadmin:latest' - Usa la imagen oficial de phpMyAdmin
  • ports: - '${FORWARD_PHPMYADMIN_PORT:-8080}:80' - Expone phpMyAdmin en el puerto 8080 de tu máquina
  • PMA_HOST: mysql - Le dice a phpMyAdmin que se conecte al servicio MySQL
  • PMA_USER y PMA_PASSWORD - Usa las credenciales de tu archivo .env
  • UPLOAD_LIMIT: 300M - Permite importar archivos SQL de hasta 300MB
  • depends_on: - mysql - Asegura que MySQL se inicie antes que phpMyAdmin

Paso 3: Ejemplo completo del archivo compose.yaml

Tu archivo debería verse así:

services:
    laravel.test:
        build:
            context: './vendor/laravel/sail/runtimes/8.5'
            dockerfile: Dockerfile
            args:
                WWWGROUP: '${WWWGROUP}'
        image: 'sail-8.5/app'
        extra_hosts:
            - 'host.docker.internal:host-gateway'
        ports:
            - '${APP_PORT:-80}:80'
            - '${VITE_PORT:-5173}:${VITE_PORT:-5173}'
        environment:
            WWWUSER: '${WWWUSER}'
            LARAVEL_SAIL: 1
            XDEBUG_MODE: '${SAIL_XDEBUG_MODE:-off}'
            XDEBUG_CONFIG: '${SAIL_XDEBUG_CONFIG:-client_host=host.docker.internal}'
            IGNITION_LOCAL_SITES_PATH: '${PWD}'
        volumes:
            - '.:/var/www/html'
        networks:
            - sail
        depends_on:
            - mysql
    mysql:
        image: 'mysql:8.4'
        ports:
            - '${FORWARD_DB_PORT:-3306}:3306'
        environment:
            MYSQL_ROOT_PASSWORD: '${DB_PASSWORD}'
            MYSQL_ROOT_HOST: '%'
            MYSQL_DATABASE: '${DB_DATABASE}'
            MYSQL_USER: '${DB_USERNAME}'
            MYSQL_PASSWORD: '${DB_PASSWORD}'
            MYSQL_ALLOW_EMPTY_PASSWORD: 1
            MYSQL_EXTRA_OPTIONS: '${MYSQL_EXTRA_OPTIONS:-}'
        volumes:
            - 'sail-mysql:/var/lib/mysql'
            - './vendor/laravel/sail/database/mysql/create-testing-database.sh:/docker-entrypoint-initdb.d/10-create-testing-database.sh'
        networks:
            - sail
        healthcheck:
            test:
                - CMD
                - mysqladmin
                - ping
                - '-p${DB_PASSWORD}'
            retries: 3
            timeout: 5s
    phpmyadmin:
        image: 'phpmyadmin:latest'
        ports:
            - '${FORWARD_PHPMYADMIN_PORT:-8080}:80'
        environment:
            PMA_HOST: mysql
            PMA_USER: '${DB_USERNAME}'
            PMA_PASSWORD: '${DB_PASSWORD}'
            UPLOAD_LIMIT: 300M
        networks:
            - sail
        depends_on:
            - mysql
networks:
    sail:
        driver: bridge
volumes:
    sail-mysql:
        driver: local

Paso 4: Reiniciar los contenedores

Si Sail ya está corriendo, necesitas reiniciarlo para que reconozca el nuevo servicio:

./vendor/bin/sail down
./vendor/bin/sail up -d

Si es la primera vez que inicias Sail:

./vendor/bin/sail up -d

El flag -d ejecuta los contenedores en segundo plano (modo detached).

Paso 5: Acceder a phpMyAdmin

Abre tu navegador y ve a:

http://localhost:8080

Credenciales de acceso:

  • Servidor: Déjalo en blanco o escribe mysql
  • Usuario: sail (o el valor de DB_USERNAME en tu .env)
  • Contraseña: password (o el valor de DB_PASSWORD en tu .env)

Personalización del puerto

Si el puerto 8080 ya está en uso o prefieres otro, puedes cambiarlo de dos formas:

Opción 1: Editar el archivo .env

Añade esta línea a tu archivo .env:

FORWARD_PHPMYADMIN_PORT=8081

Luego reinicia Sail:

./vendor/bin/sail down
./vendor/bin/sail up -d

Ahora phpMyAdmin estará en http://localhost:8081

Opción 2: Cambiar directamente en compose.yaml

Modifica la línea de puertos:

ports:
    - '8081:80'  # Cambia 8080 por el puerto que quieras

Verificar que phpMyAdmin está corriendo

Para ver todos los contenedores activos:

./vendor/bin/sail ps

Deberías ver algo como:

NAME                    IMAGE               STATUS
arena-laravel.test-1    sail-8.5/app        Up
arena-mysql-1           mysql:8.4           Up
arena-phpmyadmin-1      phpmyadmin:latest   Up

Ver los logs de phpMyAdmin

Si tienes problemas, revisa los logs:

./vendor/bin/sail logs phpmyadmin

O para seguir los logs en tiempo real:

./vendor/bin/sail logs -f phpmyadmin

Tareas comunes en phpMyAdmin

Crear una nueva tabla

  1. Selecciona tu base de datos en el panel izquierdo
  2. Haz clic en “Nueva” en la parte superior
  3. Define el nombre de la tabla y el número de columnas
  4. Configura cada columna (nombre, tipo, longitud, etc.)
  5. Haz clic en “Guardar”

Ejecutar consultas SQL

  1. Selecciona tu base de datos
  2. Haz clic en la pestaña “SQL”
  3. Escribe tu consulta
  4. Haz clic en “Continuar”

Importar una base de datos

  1. Selecciona tu base de datos
  2. Haz clic en la pestaña “Importar”
  3. Selecciona tu archivo .sql
  4. Haz clic en “Continuar”

Exportar una base de datos

  1. Selecciona tu base de datos
  2. Haz clic en la pestaña “Exportar”
  3. Elige el método (rápido o personalizado)
  4. Haz clic en “Continuar”

Solución de problemas

Error: “Cannot connect to MySQL server”

Causa: phpMyAdmin no puede conectarse a MySQL.

Solución:

  1. Verifica que MySQL esté corriendo: ./vendor/bin/sail ps
  2. Revisa las credenciales en tu archivo .env
  3. Asegúrate de que PMA_HOST sea mysql (nombre del servicio)

Error: “Access denied for user”

Causa: Las credenciales son incorrectas.

Solución:

  1. Verifica tu archivo .env:
    DB_USERNAME=sail
DB_PASSWORD=password
  2. Usa esas mismas credenciales en phpMyAdmin

Puerto 8080 ya está en uso

Causa: Otro servicio está usando el puerto 8080.

Solución:
Cambia el puerto como se explicó en la sección “Personalización del puerto”.

phpMyAdmin no aparece después de reiniciar

Causa: Error en la sintaxis del archivo compose.yaml.

Solución:

  1. Verifica la indentación (usa espacios, no tabs)
  2. Revisa que no falten dos puntos o guiones
  3. Ejecuta: ./vendor/bin/sail down y luego ./vendor/bin/sail up -d

Seguridad

En desarrollo

phpMyAdmin está bien para desarrollo local, pero ten en cuenta:

  • No expongas el puerto 8080 públicamente
  • Usa contraseñas fuertes incluso en desarrollo

En producción

NO uses phpMyAdmin en producción a menos que:

  • Esté detrás de una VPN
  • Tenga autenticación adicional
  • Use HTTPS
  • Tenga restricciones de IP

Para producción, es mejor usar herramientas de línea de comandos o servicios administrados.

Comandos útiles

# Iniciar Sail con phpMyAdmin
./vendor/bin/sail up -d

# Detener todos los servicios
./vendor/bin/sail down

# Ver logs de phpMyAdmin
./vendor/bin/sail logs phpmyadmin

# Ver todos los contenedores
./vendor/bin/sail ps

# Reiniciar solo phpMyAdmin
./vendor/bin/sail restart phpmyadmin

# Acceder a la terminal de phpMyAdmin (raramente necesario)
./vendor/bin/sail exec phpmyadmin sh

Alternativas a phpMyAdmin

Si phpMyAdmin no te convence, hay otras opciones:

Adminer

Más ligero que phpMyAdmin. Para usarlo, cambia en compose.yaml:

    adminer:
        image: 'adminer:latest'
        ports:
            - '8080:8080'
        networks:
            - sail
        depends_on:
            - mysql

TablePlus / DBeaver

Aplicaciones de escritorio que se conectan directamente a MySQL:

  • Host: localhost
  • Puerto: 3306
  • Usuario: sail
  • Contraseña: password

MySQL Workbench

Cliente oficial de MySQL con muchas características avanzadas.

Resumen rápido

  1. Edita compose.yaml y añade el servicio phpmyadmin
  2. Reinicia Sail: ./vendor/bin/sail down && ./vendor/bin/sail up -d
  3. Accede a http://localhost:8080
  4. Usa las credenciales de tu archivo .env

Guía de Comandos Básicos de Laravel Sail y Docker

Esta guía te muestra los comandos esenciales para gestionar tu entorno de desarrollo con Laravel Sail. Sail es básicamente un wrapper de Docker Compose, así que también incluimos comandos Docker útiles.

Comandos básicos de Sail

Iniciar y detener servicios

# Iniciar todos los servicios en segundo plano
./vendor/bin/sail up -d

# Iniciar y ver los logs en tiempo real
./vendor/bin/sail up

# Detener todos los servicios
./vendor/bin/sail down

# Detener y eliminar volúmenes (¡CUIDADO! Borra la base de datos)
./vendor/bin/sail down -v

# Reiniciar todos los servicios
./vendor/bin/sail restart

# Reiniciar un servicio específico
./vendor/bin/sail restart mysql
./vendor/bin/sail restart phpmyadmin

Ver el estado de los servicios

# Ver todos los contenedores y su estado
./vendor/bin/sail ps

# Ver procesos en ejecución dentro de los contenedores
./vendor/bin/sail top

Ver logs

# Ver logs de todos los servicios
./vendor/bin/sail logs

# Ver logs en tiempo real (seguir)
./vendor/bin/sail logs -f

# Ver logs de un servicio específico
./vendor/bin/sail logs mysql
./vendor/bin/sail logs phpmyadmin
./vendor/bin/sail logs laravel.test

# Ver las últimas 50 líneas de logs
./vendor/bin/sail logs --tail=50

# Combinar: seguir logs de MySQL
./vendor/bin/sail logs -f mysql

Comandos de Laravel Artisan

# Ejecutar cualquier comando Artisan
./vendor/bin/sail artisan

# Ver todas las rutas
./vendor/bin/sail artisan route:list

# Ejecutar migraciones
./vendor/bin/sail artisan migrate

# Revertir última migración
./vendor/bin/sail artisan migrate:rollback

# Resetear y ejecutar todas las migraciones
./vendor/bin/sail artisan migrate:fresh

# Ejecutar migraciones con seeders
./vendor/bin/sail artisan migrate:fresh --seed

# Crear un modelo
./vendor/bin/sail artisan make:model Post

# Crear modelo con migración y controlador
./vendor/bin/sail artisan make:model Post -mc

# Crear un controlador
./vendor/bin/sail artisan make:controller PostController

# Crear un middleware
./vendor/bin/sail artisan make:middleware CheckAge

# Limpiar caché
./vendor/bin/sail artisan cache:clear
./vendor/bin/sail artisan config:clear
./vendor/bin/sail artisan route:clear
./vendor/bin/sail artisan view:clear

# Abrir Tinker (consola interactiva)
./vendor/bin/sail artisan tinker

# Ver información de la aplicación
./vendor/bin/sail artisan about

Comandos de Composer

# Instalar dependencias
./vendor/bin/sail composer install

# Actualizar dependencias
./vendor/bin/sail composer update

# Añadir un paquete
./vendor/bin/sail composer require laravel/sanctum

# Añadir un paquete de desarrollo
./vendor/bin/sail composer require --dev laravel/pint

# Eliminar un paquete
./vendor/bin/sail composer remove nombre/paquete

# Ver paquetes instalados
./vendor/bin/sail composer show

# Autoload (regenerar archivos de autoload)
./vendor/bin/sail composer dump-autoload

Comandos de NPM

# Instalar dependencias de Node
./vendor/bin/sail npm install

# Ejecutar desarrollo (watch mode)
./vendor/bin/sail npm run dev

# Compilar para producción
./vendor/bin/sail npm run build

# Añadir un paquete
./vendor/bin/sail npm install vue

# Añadir un paquete de desarrollo
./vendor/bin/sail npm install --save-dev tailwindcss

# Actualizar paquetes
./vendor/bin/sail npm update

# Ver paquetes instalados
./vendor/bin/sail npm list

Acceso a bases de datos

# Acceder a MySQL
./vendor/bin/sail mysql

# Ejecutar un comando SQL directamente
./vendor/bin/sail mysql -e "SHOW DATABASES;"

# Importar un archivo SQL
./vendor/bin/sail mysql arena < backup.sql

# Exportar la base de datos
./vendor/bin/sail mysql arena > backup.sql

# Acceder a PostgreSQL (si lo usas)
./vendor/bin/sail psql

# Acceder a Redis (si lo usas)
./vendor/bin/sail redis

Ejecutar comandos dentro de los contenedores

# Abrir una shell en el contenedor de Laravel
./vendor/bin/sail shell

# Abrir una shell como root
./vendor/bin/sail root-shell

# Ejecutar un comando específico
./vendor/bin/sail exec laravel.test php -v

# Ejecutar comando en otro contenedor
./vendor/bin/sail exec mysql mysql --version

Ejecutar tests

# Ejecutar todos los tests
./vendor/bin/sail test

# Ejecutar tests con cobertura
./vendor/bin/sail test --coverage

# Ejecutar un test específico
./vendor/bin/sail test --filter NombreDelTest

# Ejecutar tests de un archivo
./vendor/bin/sail test tests/Feature/ExampleTest.php

# Ejecutar PHPUnit directamente
./vendor/bin/sail phpunit

Comandos de Docker (sin Sail)

A veces necesitas usar Docker directamente:

# Ver todos los contenedores (corriendo)
docker ps

# Ver todos los contenedores (incluso detenidos)
docker ps -a

# Ver imágenes descargadas
docker images

# Ver uso de recursos
docker stats

# Ver logs de un contenedor
docker logs arena-mysql-1
docker logs -f arena-mysql-1

# Detener un contenedor específico
docker stop arena-mysql-1

# Iniciar un contenedor específico
docker start arena-mysql-1

# Reiniciar un contenedor
docker restart arena-mysql-1

# Eliminar un contenedor detenido
docker rm arena-mysql-1

# Eliminar una imagen
docker rmi phpmyadmin:latest

# Ver redes de Docker
docker network ls

# Ver volúmenes de Docker
docker volume ls

# Inspeccionar un contenedor
docker inspect arena-mysql-1

# Ver procesos dentro de un contenedor
docker top arena-mysql-1

# Ejecutar comando en un contenedor
docker exec -it arena-mysql-1 bash
docker exec arena-mysql-1 mysql -u sail -ppassword -e "SHOW DATABASES;"

Limpieza y mantenimiento

# Limpiar contenedores detenidos
docker container prune

# Limpiar imágenes sin usar
docker image prune

# Limpiar volúmenes sin usar
docker volume prune

# Limpiar redes sin usar
docker network prune

# Limpiar todo (¡CUIDADO!)
docker system prune

# Limpiar todo incluyendo volúmenes
docker system prune -a --volumes

# Ver espacio usado por Docker
docker system df

Reconstruir contenedores

# Reconstruir las imágenes de Sail
./vendor/bin/sail build

# Reconstruir sin usar caché
./vendor/bin/sail build --no-cache

# Reconstruir un servicio específico
./vendor/bin/sail build laravel.test

Crear alias para Sail

En lugar de escribir ./vendor/bin/sail cada vez, crea un alias:

Para Bash

echo "alias sail='./vendor/bin/sail'" >> ~/.bashrc
source ~/.bashrc

Para Zsh

echo "alias sail='./vendor/bin/sail'" >> ~/.zshrc
source ~/.zshrc

Uso después del alias

# Ahora puedes usar simplemente:
sail up -d
sail artisan migrate
sail composer require paquete
sail npm run dev
sail test

Comandos útiles combinados

# Reiniciar todo desde cero (sin perder datos)
sail down && sail up -d

# Reiniciar y ver logs
sail down && sail up

# Limpiar caché de Laravel
sail artisan cache:clear && sail artisan config:clear && sail artisan route:clear

# Resetear base de datos con datos de prueba
sail artisan migrate:fresh --seed

# Instalar todo (Composer + NPM)
sail composer install && sail npm install

# Actualizar todo
sail composer update && sail npm update

# Compilar assets y ejecutar tests
sail npm run build && sail test

Comandos para desarrollo diario

# Iniciar el día
cd /ruta/a/tu/proyecto
sail up -d
sail npm run dev

# Durante el desarrollo
sail artisan make:model NombreModelo -mc
sail artisan migrate
sail test

# Terminar el día
sail down

Solución de problemas con comandos

Puerto ocupado

# Ver qué está usando el puerto 80
sudo lsof -i :80

# Ver qué está usando el puerto 3306
sudo lsof -i :3306

# Matar proceso en un puerto
sudo kill -9 $(sudo lsof -t -i:80)

Contenedor no inicia

# Ver logs detallados
sail logs nombre-servicio

# Reiniciar el servicio
sail restart nombre-servicio

# Reconstruir el contenedor
sail down
sail build --no-cache
sail up -d

Problemas de permisos

# Cambiar permisos de storage y bootstrap/cache
sudo chmod -R 775 storage bootstrap/cache
sudo chown -R $USER:$USER storage bootstrap/cache

# Dentro del contenedor
sail shell
chmod -R 775 storage bootstrap/cache

Base de datos no conecta

# Verificar que MySQL está corriendo
sail ps

# Ver logs de MySQL
sail logs mysql

# Reiniciar MySQL
sail restart mysql

# Acceder a MySQL para verificar
sail mysql

Limpiar todo y empezar de nuevo

# Detener y eliminar todo
sail down -v

# Limpiar Docker
docker system prune -a

# Volver a iniciar
sail up -d
sail artisan migrate

Comandos de información

# Ver versión de PHP
sail php -v

# Ver versión de Composer
sail composer --version

# Ver versión de Node
sail node --version

# Ver versión de NPM
sail npm --version

# Ver configuración de Laravel
sail artisan about

# Ver variables de entorno
sail artisan env

# Ver información de la base de datos
sail mysql -e "SELECT VERSION();"

Comandos avanzados

# Ejecutar comandos en paralelo
sail artisan queue:work &
sail npm run dev

# Exportar base de datos con timestamp
sail mysql arena > backup_$(date +%Y%m%d_%H%M%S).sql

# Monitorear logs de múltiples servicios
sail logs -f mysql & sail logs -f laravel.test

# Ejecutar comando y guardar salida
sail artisan route:list > routes.txt

# Buscar en logs
sail logs | grep "error"
sail logs mysql | grep "ERROR"

Atajos de teclado útiles

Cuando estás viendo logs con sail logs -f:

  • Ctrl + C - Detener el seguimiento de logs
  • Ctrl + Z - Pausar el proceso
  • fg - Volver al proceso pausado

Comandos para producción

Nota: Sail está diseñado para desarrollo. Para producción, usa Docker Compose directamente.

# Optimizar Laravel para producción
sail artisan config:cache
sail artisan route:cache
sail artisan view:cache
sail artisan event:cache

# Compilar assets para producción
sail npm run build

# Limpiar archivos de desarrollo
sail composer install --no-dev --optimize-autoloader
