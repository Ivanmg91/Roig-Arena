Guía de Creación de API Resources para Roig Arena

Esta guía explica cómo crear API Resources para formatear las respuestas JSON de tu API de forma consistente y controlada.

¿Qué son los API Resources?

Los API Resources son clases que transforman tus modelos Eloquent en respuestas JSON. Te permiten controlar exactamente qué datos exponer, cómo formatearlos y ocultar información sensible.

¿Por qué usar Resources?

Sin Resources (❌ Problemático)

// Controlador
public function show($id)
{
    $user = User::find($id);
    return response()->json($user); // Expone TODO, incluso password
}

// Respuesta JSON
{
    "id": 1,
    "nombre": "Juan",
    "email": "juan@example.com",
    "password": "$2y$10$...",  // ❌ Expuesto
    "remember_token": "...",    // ❌ Expuesto
    "created_at": "2026-02-19T10:30:00.000000Z"  // ❌ Formato feo
}

Con Resources (✅ Controlado)

// Controlador
public function show($id)
{
    $user = User::find($id);
    return new UserResource($user);
}

// Respuesta JSON
{
    "data": {
        "id": 1,
        "nombre": "Juan",
        "email": "juan@example.com",
        "registrado": "19/02/2026"  // ✅ Formateado
    }
}

Estructura de un Resource

Ubicación

app/Http/Resources/
├── UserResource.php
├── EventoResource.php
├── SectorResource.php
├── AsientoResource.php
├── ReservaResource.php
└── EntradaResource.php

Anatomía básica

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class EventoResource extends JsonResource
{
    /**
     * Transform the resource into an array.
     */
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'nombre' => $this->nombre,
            'fecha' => $this->fecha->format('d/m/Y'),
            'hora' => $this->hora->format('H:i'),
            'poster' => $this->poster_url,
            'descripcion' => $this->descripcion_corta,
        ];
    }
}

Resources del proyecto

1. UserResource

Propósito: Ocultar datos sensibles del usuario (password, tokens).

Ubicación: app/Http/Resources/UserResource.php

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class UserResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'nombre' => $this->nombre,
            'apellido' => $this->apellido,
            'nombre_completo' => $this->nombre . ' ' . $this->apellido,
            'email' => $this->email,
            'is_admin' => $this->is_admin,
            'registrado' => $this->created_at->format('d/m/Y'),
        ];
    }
}

Uso en controlador:

// AuthController
public function user(Request $request)
{
    return new UserResource($request->user());
}

2. EventoResource

Propósito: Formatear eventos con información adicional.

Ubicación: app/Http/Resources/EventoResource.php

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class EventoResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'nombre' => $this->nombre,
            'descripcion_corta' => $this->descripcion_corta,
            'descripcion_larga' => $this->when($request->routeIs('eventos.show'), $this->descripcion_larga),
            'poster' => $this->poster_url,
            'fecha' => $this->fecha->format('d/m/Y'),
            'hora' => $this->hora ? $this->hora->format('H:i') : null,
            'sectores' => SectorResource::collection($this->whenLoaded('sectores')),
            'precios' => PrecioResource::collection($this->whenLoaded('precios')),
            'asientos_disponibles' => $this->when(
                $request->routeIs('eventos.show'),
                fn() => $this->totalAsientosDisponibles()
            ),
            'entradas_vendidas' => $this->when(
                $request->routeIs('eventos.show'),
                fn() => $this->totalEntradasVendidas()
            ),
        ];
    }
}

Características:

  • when() - Incluir campo condicionalmente
  • whenLoaded() - Solo si la relación está cargada (eager loading)
  • Formateo de fechas
  • Relaciones anidadas

Uso en controlador:

// EventoController
public function index()
{
    $eventos = Evento::futuros()->with('precios.sector')->get();
    return EventoResource::collection($eventos);
}

public function show($id)
{
    $evento = Evento::with('precios.sector')->findOrFail($id);
    return new EventoResource($evento);
}

3. SectorResource

Propósito: Formatear sectores con información de asientos.

Ubicación: app/Http/Resources/SectorResource.php

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class SectorResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'nombre' => $this->nombre,
            'descripcion' => $this->descripcion,
            'activo' => $this->activo,
            'total_asientos' => $this->when(
                $this->relationLoaded('asientos'),
                fn() => $this->asientos->count()
            ),
            'precio' => $this->when(
                isset($this->pivot),
                fn() => [
                    'valor' => number_format($this->pivot->precio, 2, ',', '.') . ' €',
                    'disponible' => $this->pivot->disponible,
                ]
            ),
        ];
    }
}

Características:

  • Acceso a datos de tabla pivot
  • Formateo de precios
  • Contador condicional

4. AsientoResource

Propósito: Formatear asientos con estado de disponibilidad.

Ubicación: app/Http/Resources/AsientoResource.php

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class AsientoResource extends JsonResource
{
    protected $eventoId;

    public function __construct($resource, $eventoId = null)
    {
        parent::__construct($resource);
        $this->eventoId = $eventoId;
    }

    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'sector' => $this->sector->nombre,
            'sector_id' => $this->sector_id,
            'fila' => $this->fila,
            'numero' => $this->numero,
            'nombre_completo' => $this->nombreCompleto(),
            'disponible' => $this->when(
                $this->eventoId,
                fn() => $this->estaDisponibleParaEvento($this->eventoId)
            ),
        ];
    }

    public static function collectionWithEvento($resource, $eventoId)
    {
        return $resource->map(function ($item) use ($eventoId) {
            return new static($item, $eventoId);
        });
    }
}

Características:

  • Constructor personalizado para pasar datos adicionales
  • Método estático para colecciones con contexto
  • Estado dinámico basado en evento

Uso en controlador:

// AsientoController
public function porEvento($eventoId)
{
    $asientos = Asiento::with('sector')->get();
    return AsientoResource::collectionWithEvento($asientos, $eventoId);
}

5. PrecioResource

Propósito: Formatear precios con información del sector.

Ubicación: app/Http/Resources/PrecioResource.php

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class PrecioResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'sector' => new SectorResource($this->whenLoaded('sector')),
            'precio' => number_format($this->precio, 2, ',', '.') . ' €',
            'precio_raw' => $this->precio,
            'disponible' => $this->disponible,
        ];
    }
}

6. ReservaResource

Propósito: Formatear reservas con temporizador.

Ubicación: app/Http/Resources/ReservaResource.php

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class ReservaResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'evento' => [
                'id' => $this->evento->id,
                'nombre' => $this->evento->nombre,
                'fecha' => $this->evento->fecha->format('d/m/Y'),
                'hora' => $this->evento->hora ? $this->evento->hora->format('H:i') : null,
            ],
            'asiento' => [
                'id' => $this->asiento->id,
                'nombre' => $this->asiento->nombreCompleto(),
                'sector' => $this->asiento->sector->nombre,
            ],
            'precio' => number_format(
                $this->evento->precioDelSector($this->asiento->sector_id)?->precio ?? 0,
                2, ',', '.'
            ) . ' €',
            'estado' => $this->estado,
            'tiempo_restante_minutos' => $this->tiempoRestante(),
            'expira_en' => $this->reservado_hasta?->format('d/m/Y H:i:s'),
            'expirado' => $this->haExpirado(),
        ];
    }
}

Características:

  • Relaciones anidadas con formato personalizado
  • Cálculos dinámicos (tiempo restante)
  • Información de expiración

7. EntradaResource

Propósito: Formatear entradas con código QR e información completa.

Ubicación: app/Http/Resources/EntradaResource.php

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class EntradaResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'codigo_qr' => $this->codigo_qr,
            'evento' => [
                'id' => $this->evento->id,
                'nombre' => $this->evento->nombre,
                'fecha' => $this->evento->fecha->format('d/m/Y'),
                'hora' => $this->evento->hora ? $this->evento->hora->format('H:i') : null,
                'poster' => $this->evento->poster_url,
            ],
            'asiento' => [
                'id' => $this->asiento->id,
                'nombre' => $this->asiento->nombreCompleto(),
                'sector' => $this->asiento->sector->nombre,
                'fila' => $this->asiento->fila,
                'numero' => $this->asiento->numero,
            ],
            'precio_pagado' => number_format($this->precio_pagado, 2, ',', '.') . ' €',
            'comprador' => $this->user->nombre . ' ' . $this->user->apellido,
            'fecha_compra' => $this->created_at->format('d/m/Y H:i'),
            'valida' => $this->esValida(),
        ];
    }
}

Métodos útiles de Resources

when() - Incluir condicionalmente

'campo' => $this->when($condicion, $valor)

// Ejemplo
'descripcion_larga' => $this->when(
    $request->routeIs('eventos.show'),
    $this->descripcion_larga
)

whenLoaded() - Solo si está cargada la relación

'sectores' => SectorResource::collection($this->whenLoaded('sectores'))

merge() - Combinar arrays

return array_merge(parent::toArray($request), [
    'campo_extra' => 'valor',
]);

with() - Añadir metadatos

public function with(Request $request): array
{
    return [
        'version' => '1.0',
        'timestamp' => now()->toIso8601String(),
    ];
}

Resource Collections

Para colecciones de recursos, Laravel automáticamente las envuelve en un array data:

// Controlador
return EventoResource::collection($eventos);

// Respuesta
{
    "data": [
        { "id": 1, "nombre": "Evento 1" },
        { "id": 2, "nombre": "Evento 2" }
    ]
}

Personalizar el wrapper

// En el Resource
public static $wrap = 'eventos'; // Cambia 'data' por 'eventos'

// O desactivarlo
public static $wrap = null;

Añadir metadatos a colecciones

Crear un Resource Collection personalizado:

// app/Http/Resources/EventoCollection.php
<?php

namespace App\Http\Resources;

use Illuminate\Http\Resources\Json\ResourceCollection;

class EventoCollection extends ResourceCollection
{
    public function toArray(Request $request): array
    {
        return [
            'data' => $this->collection,
            'meta' => [
                'total' => $this->collection->count(),
                'fecha_consulta' => now()->format('d/m/Y H:i'),
            ],
        ];
    }
}

Uso:

return new EventoCollection($eventos);

Apéndice: API Resources en Laravel para Principiantes

Esta guía te ayudará a entender qué son los API Resources, cómo funcionan y el vocabulario específico que usa Laravel para transformar datos.

¿Qué son los API Resources?

Los API Resources son clases que transforman modelos Eloquent en respuestas JSON. Actúan como una capa de presentación entre tus datos y el cliente de la API.

Analogía simple

Imagina un traductor en un restaurante internacional:

  • Modelo = El plato en la cocina (datos crudos)
  • Resource = El traductor (transforma y presenta)
  • JSON = El menú traducido para el cliente (datos formateados)

El traductor decide qué información mostrar, en qué idioma (formato) y qué ocultar (ingredientes secretos = datos sensibles).

¿Por qué usar Resources?

Problema sin Resources

// Controlador
$user = User::find(1);
return response()->json($user);

// Respuesta JSON - Expone TODO
{
    "id": 1,
    "nombre": "Juan",
    "email": "juan@example.com",
    "password": "$2y$10$abcd...",      // ❌ PELIGRO
    "remember_token": "xyz123...",      // ❌ PELIGRO
    "created_at": "2026-02-19T10:30:00.000000Z",  // ❌ Feo
    "updated_at": "2026-02-19T10:30:00.000000Z"
}

Solución con Resources

// Controlador
$user = User::find(1);
return new UserResource($user);

// Respuesta JSON - Controlada
{
    "data": {
        "id": 1,
        "nombre": "Juan",
        "email": "juan@example.com",
        "registrado": "19/02/2026"     // ✅ Formateado
    }
}

Ventajas:

  • ✅ Control total sobre qué exponer
  • ✅ Ocultar datos sensibles
  • ✅ Formatear fechas, números, etc.
  • ✅ Respuestas consistentes
  • ✅ Fácil de mantener

Estructura de un Resource

Ubicación

app/Http/Resources/
├── UserResource.php
├── EventoResource.php
├── SectorResource.php
└── ...

Anatomía básica

<?php

// 1. NAMESPACE
namespace App\Http\Resources;

// 2. IMPORTS
use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

// 3. CLASE - Extiende JsonResource
class EventoResource extends JsonResource
{
    /**
     * 4. MÉTODO toArray - Define la estructura JSON
     */
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'nombre' => $this->nombre,
            'fecha' => $this->fecha->format('d/m/Y'),
        ];
    }
}

Imports necesarios

1. JsonResource (obligatorio)

use Illuminate\Http\Resources\Json\JsonResource;

¿Qué es? La clase base de todos los Resources.
¿Para qué? Proporciona métodos para transformar modelos a JSON.

2. Request

use Illuminate\Http\Request;

¿Qué es? Objeto de la petición HTTP.
¿Para qué? Acceder a datos de la petición para condicionales.

3. ResourceCollection (opcional)

use Illuminate\Http\Resources\Json\ResourceCollection;

¿Qué es? Clase base para colecciones personalizadas.
¿Para qué? Cuando necesitas añadir metadatos a colecciones.

Método principal: toArray()

El método toArray() define cómo se transforma el modelo a JSON.

Acceso a propiedades del modelo

Dentro del Resource, $this hace referencia al modelo:

public function toArray(Request $request): array
{
    return [
        'id' => $this->id,              // Acceso directo
        'nombre' => $this->nombre,
        'email' => $this->email,
        
        // Métodos del modelo
        'nombre_completo' => $this->nombreCompleto(),
        
        // Relaciones
        'sector' => $this->sector->nombre,
        
        // Formateo
        'fecha' => $this->fecha->format('d/m/Y'),
    ];
}

Tipos de datos que puedes devolver

return [
    // Valores simples
    'id' => $this->id,                    // Integer
    'nombre' => $this->nombre,            // String
    'activo' => $this->activo,            // Boolean
    
    // Formateo
    'precio' => number_format($this->precio, 2, ',', '.') . ' €',
    'fecha' => $this->fecha->format('d/m/Y'),
    
    // Arrays
    'tags' => ['tag1', 'tag2'],
    
    // Objetos anidados
    'sector' => [
        'id' => $this->sector->id,
        'nombre' => $this->sector->nombre,
    ],
    
    // Otros Resources
    'sector' => new SectorResource($this->sector),
    
    // Colecciones de Resources
    'asientos' => AsientoResource::collection($this->asientos),
];

Métodos útiles de Resources

when() - Incluir condicionalmente

'campo' => $this->when($condicion, $valor)

Ejemplo 1: Según la ruta

'descripcion_larga' => $this->when(
    $request->routeIs('eventos.show'),
    $this->descripcion_larga
)
// Solo incluye descripcion_larga en la ruta eventos.show

Ejemplo 2: Según el usuario

'email' => $this->when(
    $request->user()->isAdmin(),
    $this->email
)
// Solo admins ven el email

Ejemplo 3: Con closure

'total_asientos' => $this->when(
    $request->has('incluir_total'),
    fn() => $this->asientos->count()
)
// Solo si se pide explícitamente

whenLoaded() - Solo si la relación está cargada

'sectores' => SectorResource::collection($this->whenLoaded('sectores'))

¿Por qué? Evita el problema N+1. Solo incluye la relación si se hizo eager loading.

Ejemplo:

// Controlador CON eager loading
$evento = Evento::with('sectores')->find(1);
return new EventoResource($evento);
// Respuesta incluye 'sectores'

// Controlador SIN eager loading
$evento = Evento::find(1);
return new EventoResource($evento);
// Respuesta NO incluye 'sectores'

whenPivotLoaded() - Datos de tabla pivot

'precio' => $this->whenPivotLoaded('precios', function () {
    return $this->pivot->precio;
})

Uso: Cuando tienes relaciones muchos a muchos con datos extra en la tabla pivot.

mergeWhen() - Combinar arrays condicionalmente

return array_merge([
    'id' => $this->id,
    'nombre' => $this->nombre,
], $this->mergeWhen($request->user()->isAdmin(), [
    'created_at' => $this->created_at,
    'updated_at' => $this->updated_at,
]));

Resource Collections

Colección automática

// Controlador
$eventos = Evento::all();
return EventoResource::collection($eventos);

// Respuesta
{
    "data": [
        { "id": 1, "nombre": "Evento 1" },
        { "id": 2, "nombre": "Evento 2" }
    ]
}

Laravel automáticamente envuelve la colección en data.

Colección personalizada

Cuando necesitas añadir metadatos:

// app/Http/Resources/EventoCollection.php
<?php

namespace App\Http\Resources;

use Illuminate\Http\Resources\Json\ResourceCollection;

class EventoCollection extends ResourceCollection
{
    public function toArray(Request $request): array
    {
        return [
            'data' => $this->collection,
            'meta' => [
                'total' => $this->collection->count(),
                'fecha_consulta' => now()->format('d/m/Y H:i'),
            ],
        ];
    }
}

Uso:

return new EventoCollection($eventos);

// Respuesta
{
    "data": [...],
    "meta": {
        "total": 10,
        "fecha_consulta": "19/02/2026 10:30"
    }
}

Cambiar ‘data’ por otro nombre

// En el Resource
class EventoResource extends JsonResource
{

// Respuesta
{
    "eventos": { ... }
}

Desactivar el wrapper

public static $wrap = null;

Configurar globalmente

En AppServiceProvider:

public function boot(): void

Añadir metadatos

Método with()

class EventoResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'nombre' => $this->nombre,
        ];
    }

Relaciones anidadas

Opción 1: Objeto anidado simple

public function toArray(Request $request): array
{
    return [
        'id' => $this->id,
        'evento' => [
            'id' => $this->evento->id,
            'nombre' => $this->evento->nombre,
        ],
    ];

A veces necesitas pasar contexto extra:

// Resource con constructor personalizado
class AsientoResource extends JsonResource
{
    protected $eventoId;

    public function __construct($resource, $eventoId = null)
    {
        parent::__construct($resource);
        $this->eventoId = $eventoId;
    }

    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'disponible' => $this->eventoId

Patrones comunes

Ocultar campos sensibles

class UserResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'nombre' => $this->nombre,
            'email' => $this->email,
            // NO incluir: password, remember_token, etc.
        ];
    }
}

Campos solo para admins

public function toArray(Request $request): array
{
    return [
        'id' => $this->id,
        'nombre' => $this->nombre,
        
        // Solo admins ven estos campos
        'created_at' => $this->when(
            $request->user()?->isAdmin(),
            $this->created_at
        ),
        'updated_at' => $this->when(
            $request->user()?->isAdmin(),
            $this->updated_at
        ),
    ];
}

Información calculada

public function toArray(Request $request): array
{
    return [
        'id' => $this->id,
        'nombre' => $this->nombre,
        
        // Cálculos
        'total_asientos' => $this->asientos->count(),

Diferentes vistas del mismo recurso

public function toArray(Request $request): array
{
    $base = [
        'id' => $this->id,
        'nombre' => $this->nombre,
    ];
    
    // Vista completa
    if ($request->routeIs('eventos.show')) {
        return array_merge($base, [
            'descripcion_larga' => $this->descripcion_larga,
            'sectores' => SectorResource::collection($this->sectores),
        ]);
    }
    
    // Vista resumida
    return $base;
}

Buenas prácticas

✅ Hacer

// Formatear fechas consistentemente
'fecha' => $this->fecha->format('d/m/Y')

// Ocultar campos sensibles
// NO incluir password, tokens, etc.

// Usar whenLoaded para relaciones
'sectores' => SectorResource::collection($this->whenLoaded('sectores'))

// Formatear precios
'precio' => number_format($this->precio, 2, ',', '.') . ' €'

// Nombres descriptivos
'nombre_completo' => $this->nombre . ' ' . $this->apellido

// Incluir solo lo necesario
return [
    'id' => $this->id,
    'nombre' => $this->nombre,
    // Solo lo que el frontend necesita
];

❌ Evitar

// No devolver todo el modelo
return $this->resource; // ❌

// No hacer queries dentro del Resource
'total' => Asiento::where('sector_id', $this->id)->count(); // ❌
// Mejor: usar withCount en el controlador

// No exponer datos sensibles
'password' => $this->password, // ❌

// No incluir relaciones sin whenLoaded
'sectores' => SectorResource::collection($this->sectores), // ❌ N+1
// Mejor:
'sectores' => SectorResource::collection($this->whenLoaded('sectores')),

// No formatear en el controlador
// Esa es la responsabilidad del Resource

Errores comunes

Error: “Trying to get property of non-object”

Causa: Intentas acceder a una relación que no está cargada.

Solución:

// ❌ Mal
'sector' => $this->sector->nombre,

// ✅ Bien
'sector' => $this->whenLoaded('sector', fn() => $this->sector->nombre),

Error: N+1 queries

Causa: No usas whenLoaded() y accedes a relaciones.

Solución:

// Controlador
$eventos = Evento::with('sectores')->get(); // Eager loading

// Resource
'sectores' => SectorResource::collection($this->whenLoaded('sectores'))

Error: “Call to undefined method”

Causa: Intentas usar un método que no existe en el modelo.

Solución: Verifica que el método esté definido en el modelo.
