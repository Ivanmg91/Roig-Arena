Guía de Creación de Controladores API para Roig Arena

Esta guía explica cómo crear los controladores necesarios para el sistema de venta de entradas, siguiendo las mejores prácticas de Laravel.

¿Qué son los controladores?

Los controladores son clases que manejan las peticiones HTTP y contienen la lógica para responder a las acciones del usuario. En una API REST, cada controlador gestiona un recurso específico (eventos, sectores, reservas, etc.).

Estructura de controladores para el proyecto

Controladores que vamos a crear:

  1. AuthController - Registro, login, logout
  2. EventoController - CRUD de eventos (admin) + listado público
  3. SectorController - Gestión de sectores (admin)
  4. AsientoController - Consulta de asientos disponibles
  5. ReservaController - Reservar asientos (carrito)
  6. CompraController - Confirmar compra y generar entrada
  7. EntradaController - Ver mis entradas

Principios de diseño

Controladores delgados (Thin Controllers)

  • La lógica compleja va en Services
  • Los controladores solo coordinan y responden
  • Validaciones en Form Requests (opcional) o en el controlador

Respuestas consistentes

// Éxito
return response()->json(['data' => $data], 200);

// Error
return response()->json(['error' => 'Mensaje'], 400);

// Creado
return response()->json(['data' => $data], 201);

Códigos HTTP

  • 200 OK - Éxito
  • 201 Created - Recurso creado
  • 400 Bad Request - Error de validación
  • 401 Unauthorized - No autenticado
  • 403 Forbidden - Sin permisos
  • 404 Not Found - No encontrado
  • 500 Internal Server Error - Error del servidor

Controlador 1: AuthController

Propósito

Gestionar autenticación de usuarios (registro, login, logout).

Ubicación

app/Http/Controllers/Auth/AuthController.php

Rutas que maneja

POST /api/register
POST /api/login
POST /api/logout (protegida)
GET  /api/user (protegida)

Código completo

<?php

namespace App\Http\Controllers\Auth;

use App\Http\Controllers\Controller;
use App\Models\User;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Hash;
use Illuminate\Validation\ValidationException;

class AuthController extends Controller
{
    /**
     * Registrar un nuevo usuario
     */
    public function register(Request $request)
    {
        $request->validate([
            'nombre' => 'required|string|max:255',
            'apellido' => 'required|string|max:255',
            'email' => 'required|string|email|max:255|unique:users',
            'password' => 'required|string|min:8|confirmed',
        ]);

        $user = User::create([
            'nombre' => $request->nombre,
            'apellido' => $request->apellido,
            'email' => $request->email,
            'password' => Hash::make($request->password),
            'is_admin' => false,
        ]);

        $token = $user->createToken('auth-token')->plainTextToken;

        return response()->json([
            'user' => $user,
            'token' => $token,
            'token_type' => 'Bearer',
        ], 201);
    }

    /**
     * Iniciar sesión
     */
    public function login(Request $request)
    {
        $request->validate([
            'email' => 'required|email',
            'password' => 'required',
        ]);

        $user = User::where('email', $request->email)->first();

        if (!$user || !Hash::check($request->password, $user->password)) {
            throw ValidationException::withMessages([
                'email' => ['Las credenciales son incorrectas.'],
            ]);
        }

        // Eliminar tokens anteriores (opcional)
        $user->tokens()->delete();

        $token = $user->createToken('auth-token')->plainTextToken;

        return response()->json([
            'user' => $user,
            'token' => $token,
            'token_type' => 'Bearer',
        ]);
    }

    /**
     * Cerrar sesión
     */
    public function logout(Request $request)
    {
        $request->user()->currentAccessToken()->delete();

        return response()->json([
            'message' => 'Sesión cerrada correctamente',
        ]);
    }

    /**
     * Obtener usuario autenticado
     */
    public function user(Request $request)
    {
        return response()->json([
            'user' => $request->user(),
        ]);
    }
}

Controlador 2: EventoController

Propósito

  • Público: Listar eventos futuros con precios
  • Admin: CRUD completo de eventos

Ubicación

app/Http/Controllers/EventoController.php

Rutas que maneja

GET    /api/eventos              - Listar eventos futuros (público)
GET    /api/eventos/{id}         - Ver detalle de evento (público)
POST   /api/admin/eventos        - Crear evento (admin)
PUT    /api/admin/eventos/{id}   - Actualizar evento (admin)
DELETE /api/admin/eventos/{id}   - Eliminar evento (admin)

Código completo

<?php

namespace App\Http\Controllers;

use App\Models\Evento;
use Illuminate\Http\Request;

class EventoController extends Controller
{
    /**
     * Listar eventos futuros (público)
     */
    public function index()
    {
        $eventos = Evento::futuros()
            ->with(['precios.sector'])
            ->get();

        return response()->json([
            'data' => $eventos,
        ]);
    }

    /**
     * Ver detalle de un evento (público)
     */
    public function show($id)
    {
        $evento = Evento::with(['precios.sector'])
            ->findOrFail($id);

        return response()->json([
            'data' => [
                'evento' => $evento,
                'sectores_disponibles' => $evento->sectoresDisponibles(),
                'asientos_disponibles' => $evento->totalAsientosDisponibles(),
                'entradas_vendidas' => $evento->totalEntradasVendidas(),
            ],
        ]);
    }

    /**
     * Crear evento (admin)
     */
    public function store(Request $request)
    {
        $request->validate([
            'nombre' => 'required|string|max:255',
            'descripcion_corta' => 'required|string|max:255',
            'descripcion_larga' => 'required|string',
            'poster_url' => 'nullable|url',
            'fecha' => 'required|date|unique:eventos,fecha',
            'hora' => 'required|date_format:H:i',
        ]);

        $evento = Evento::create($request->all());

        return response()->json([
            'data' => $evento,
            'message' => 'Evento creado correctamente',
        ], 201);
    }

    /**
     * Actualizar evento (admin)
     */
    public function update(Request $request, $id)
    {
        $evento = Evento::findOrFail($id);

        $request->validate([
            'nombre' => 'sometimes|string|max:255',
            'descripcion_corta' => 'sometimes|string|max:255',
            'descripcion_larga' => 'sometimes|string',
            'poster_url' => 'nullable|url',
            'fecha' => 'sometimes|date|unique:eventos,fecha,' . $id,
            'hora' => 'sometimes|date_format:H:i',
        ]);

        $evento->update($request->all());

        return response()->json([
            'data' => $evento,
            'message' => 'Evento actualizado correctamente',
        ]);
    }

    /**
     * Eliminar evento (admin)
     */
    public function destroy($id)
    {
        $evento = Evento::findOrFail($id);
        
        // Verificar que no tenga entradas vendidas
        if ($evento->totalEntradasVendidas() > 0) {
            return response()->json([
                'error' => 'No se puede eliminar un evento con entradas vendidas',
            ], 400);
        }

        $evento->delete();

        return response()->json([
            'message' => 'Evento eliminado correctamente',
        ]);
    }
}

Controlador 3: SectorController

Propósito

Gestionar sectores del estadio (solo admin).

Ubicación

app/Http/Controllers/SectorController.php

Rutas que maneja

GET    /api/sectores             - Listar sectores activos (público)
POST   /api/admin/sectores       - Crear sector (admin)
PUT    /api/admin/sectores/{id}  - Actualizar sector (admin)
DELETE /api/admin/sectores/{id}  - Eliminar sector (admin)

Código completo

<?php

namespace App\Http\Controllers;

use App\Models\Sector;
use Illuminate\Http\Request;

class SectorController extends Controller
{
    /**
     * Listar sectores activos (público)
     */
    public function index()
    {
        $sectores = Sector::activos()
            ->withCount('asientos')
            ->get();

        return response()->json([
            'data' => $sectores,
        ]);
    }

    /**
     * Crear sector (admin)
     */
    public function store(Request $request)
    {
        $request->validate([
            'nombre' => 'required|string|max:255|unique:sectores',
            'descripcion' => 'nullable|string',
            'activo' => 'boolean',
        ]);

        $sector = Sector::create($request->all());

        return response()->json([
            'data' => $sector,
            'message' => 'Sector creado correctamente',
        ], 201);
    }

    /**
     * Actualizar sector (admin)
     */
    public function update(Request $request, $id)
    {
        $sector = Sector::findOrFail($id);

        $request->validate([
            'nombre' => 'sometimes|string|max:255|unique:sectores,nombre,' . $id,
            'descripcion' => 'nullable|string',
            'activo' => 'boolean',
        ]);

        $sector->update($request->all());

        return response()->json([
            'data' => $sector,
            'message' => 'Sector actualizado correctamente',
        ]);
    }

    /**
     * Eliminar sector (admin)
     */
    public function destroy($id)
    {
        $sector = Sector::findOrFail($id);
        
        // Verificar que no tenga asientos
        if ($sector->totalAsientos() > 0) {
            return response()->json([
                'error' => 'No se puede eliminar un sector con asientos',
            ], 400);
        }

        $sector->delete();

        return response()->json([
            'message' => 'Sector eliminado correctamente',
        ]);
    }
}

Controlador 4: AsientoController

Propósito

Consultar asientos disponibles para un evento.

Ubicación

app/Http/Controllers/AsientoController.php

Rutas que maneja

GET /api/eventos/{eventoId}/asientos - Asientos disponibles por evento
GET /api/eventos/{eventoId}/sectores/{sectorId}/asientos - Asientos de un sector

Código completo

<?php

namespace App\Http\Controllers;

use App\Models\Evento;
use App\Models\Sector;
use App\Models\Asiento;
use Illuminate\Http\Request;

class AsientoController extends Controller
{
    /**
     * Obtener asientos disponibles de un evento
     */
    public function porEvento($eventoId)
    {
        $evento = Evento::findOrFail($eventoId);
        
        // Obtener sectores disponibles
        $sectoresDisponibles = $evento->sectoresDisponibles()->pluck('id');
        
        // Obtener asientos de esos sectores
        $asientos = Asiento::whereIn('sector_id', $sectoresDisponibles)
            ->with('sector')
            ->get()
            ->map(function ($asiento) use ($eventoId) {
                return [
                    'id' => $asiento->id,
                    'sector' => $asiento->sector->nombre,
                    'fila' => $asiento->fila,
                    'numero' => $asiento->numero,
                    'disponible' => $asiento->estaDisponibleParaEvento($eventoId),
                    'precio' => $asiento->sector->precios()
                        ->where('evento_id', $eventoId)
                        ->first()?->precio,
                ];
            });

        return response()->json([
            'data' => $asientos,
        ]);
    }

    /**
     * Obtener asientos de un sector específico para un evento
     */
    public function porSector($eventoId, $sectorId)
    {
        $evento = Evento::findOrFail($eventoId);
        $sector = Sector::findOrFail($sectorId);
        
        // Verificar que el sector esté disponible para el evento
        if (!$evento->sectorEstaDisponible($sectorId)) {
            return response()->json([
                'error' => 'El sector no está disponible para este evento',
            ], 400);
        }

        $asientos = $sector->asientos()
            ->get()
            ->map(function ($asiento) use ($eventoId) {
                return [
                    'id' => $asiento->id,
                    'fila' => $asiento->fila,
                    'numero' => $asiento->numero,
                    'disponible' => $asiento->estaDisponibleParaEvento($eventoId),
                ];
            });

        $precio = $evento->precioDelSector($sectorId);

        return response()->json([
            'data' => [
                'sector' => $sector,
                'precio' => $precio?->precio,
                'asientos' => $asientos,
            ],
        ]);
    }
}

Controlador 5: ReservaController

Propósito

Gestionar el carrito de compras (reservas temporales).

Ubicación

app/Http/Controllers/ReservaController.php

Rutas que maneja

POST   /api/reservas           - Reservar asiento (añadir al carrito)
GET    /api/reservas           - Ver mis reservas activas
DELETE /api/reservas/{id}      - Cancelar reserva

Código completo

<?php

namespace App\Http\Controllers;

use App\Http\Resources\ReservaResource;
use App\Services\ReservaService;
use Illuminate\Http\Request;

class ReservaController extends Controller
{
    /**
     * Reservar un asiento (añadir al carrito)
     */
    public function store(Request $request, ReservaService $service)
    {
        $request->validate([
            'evento_id' => 'required|exists:eventos,id',
            'asiento_id' => 'required|exists:asientos,id',
        ]);

        try {
            $reserva = $service->reservarAsiento(
                $request->evento_id,
                $request->asiento_id,
                auth()->id()
            );

            return response()->json([
                'data' => new ReservaResource($reserva),
                'message' => 'Asiento reservado por 15 minutos',
            ], 201);
            
        } catch (\Exception $e) {
            return response()->json([
                'error' => $e->getMessage(),
            ], 400);
        }
    }

    /**
     * Ver mis reservas activas
     */
    public function index(Request $request, ReservaService $service)
    {
        $reservas = $service->obtenerReservasActivas(auth()->id());

        return ReservaResource::collection($reservas);
    }

    /**
     * Cancelar una reserva
     */
    public function destroy($id, ReservaService $service)
    {
        try {
            $service->cancelarReserva($id, auth()->id());

            return response()->json([
                'message' => 'Reserva cancelada correctamente',
            ]);
            
        } catch (\Exception $e) {
            return response()->json([
                'error' => $e->getMessage(),
            ], 400);
        }
    }
}

Nota: Este controlador usa ReservaService para encapsular la lógica de negocio compleja (verificaciones, transacciones). El controlador se mantiene delgado (15 líneas por método).

Controlador 6: CompraController

Propósito

Confirmar la compra y generar la entrada con código QR.

Ubicación

app/Http/Controllers/CompraController.php

Rutas que maneja

POST /api/compras - Confirmar compra de reservas

Código completo

<?php

namespace App\Http\Controllers;

use App\Http\Resources\EntradaResource;
use App\Services\CompraService;
use Illuminate\Http\Request;

class CompraController extends Controller
{
    /**
     * Confirmar compra de reservas
     */
    public function store(Request $request, CompraService $service)
    {
        $request->validate([
            'reservas' => 'required|array|min:1',
            'reservas.*' => 'exists:estado_asientos,id',
        ]);

        try {
            $entradas = $service->procesarCompra(
                $request->reservas,
                auth()->id()
            );

            return response()->json([
                'data' => EntradaResource::collection($entradas),
                'message' => 'Compra realizada correctamente',
            ], 201);
            
        } catch (\Exception $e) {
            return response()->json([
                'error' => $e->getMessage(),
            ], 400);
        }
    }
}

Nota: Este controlador usa CompraService para manejar la lógica compleja de compra (transacciones DB, validaciones, creación de entradas). El controlador se mantiene delgado (15 líneas).

Controlador 7: EntradaController

Propósito

Ver las entradas compradas por el usuario.

Ubicación

app/Http/Controllers/EntradaController.php

Rutas que maneja

GET /api/entradas     - Mis entradas
GET /api/entradas/{id} - Detalle de entrada

Código completo

<?php

namespace App\Http\Controllers;

use App\Models\Entrada;
use Illuminate\Http\Request;

class EntradaController extends Controller
{
    /**
     * Listar mis entradas
     */
    public function index(Request $request)
    {
        $entradas = $request->user()
            ->entradas()
            ->with(['evento', 'asiento.sector'])
            ->latest()
            ->get()
            ->map(function ($entrada) {
                return [
                    'id' => $entrada->id,
                    'codigo_qr' => $entrada->codigo_qr,
                    'evento' => $entrada->evento->nombre,
                    'fecha' => $entrada->evento->fecha->format('d/m/Y'),
                    'hora' => $entrada->evento->hora,
                    'asiento' => $entrada->asiento->nombreCompleto(),
                    'precio' => $entrada->precioFormateado(),
                    'valida' => $entrada->esValida(),
                ];
            });

        return response()->json([
            'data' => $entradas,
        ]);
    }

    /**
     * Ver detalle de una entrada
     */
    public function show($id)
    {
        $entrada = Entrada::where('id', $id)
            ->where('user_id', auth()->id())
            ->with(['evento', 'asiento.sector'])
            ->firstOrFail();

        return response()->json([
            'data' => $entrada->informacionCompleta(),
        ]);
    }
}

Apéndice: Controladores en Laravel para Principiantes

Esta guía te ayudará a entender qué son los controladores, cómo funcionan y el vocabulario específico que usa Laravel para trabajar con ellos.

¿Qué son los controladores?

Los controladores son clases PHP que manejan las peticiones HTTP y deciden qué responder. Son el “cerebro” que coordina entre las rutas, los modelos y las vistas/respuestas.

Analogía simple

Imagina un restaurante:

  • Rutas = El menú (qué platos puedes pedir)
  • Controlador = El camarero (recibe tu pedido y lo coordina)
  • Modelo = La cocina (prepara los datos)
  • Respuesta = El plato servido (lo que recibes)

Cuando pides “GET /api/eventos”, la ruta llama al controlador, el controlador pide datos al modelo, y devuelve una respuesta JSON.

¿Por qué usar controladores?

Sin controladores (❌ Problemático)

// routes/api.php
Route::get('/eventos', function () {
    $eventos = Evento::all();
    return response()->json($eventos);
});

Route::post('/eventos', function (Request $request) {
    $request->validate([...]);
    $evento = Evento::create($request->all());
    return response()->json($evento);
});
// Todo mezclado en las rutas

Con controladores (✅ Organizado)

// routes/api.php
Route::get('/eventos', [EventoController::class, 'index']);
Route::post('/eventos', [EventoController::class, 'store']);

// app/Http/Controllers/EventoController.php
class EventoController extends Controller
{
    public function index() { /* lógica aquí */ }
    public function store(Request $request) { /* lógica aquí */ }
}

Ventajas:

  • ✅ Código organizado y reutilizable
  • ✅ Fácil de mantener y testear
  • ✅ Separación de responsabilidades
  • ✅ Lógica agrupada por recurso

Estructura de un controlador

Ubicación

app/Http/Controllers/
├── Controller.php              # Controlador base
├── EventoController.php        # Controlador de eventos
├── SectorController.php        # Controlador de sectores
└── Auth/
    └── AuthController.php      # Controlador de autenticación

Anatomía de un controlador

<?php

// 1. NAMESPACE - Ubicación del controlador
namespace App\Http\Controllers;

// 2. IMPORTS - Clases que necesita
use App\Models\Evento;
use Illuminate\Http\Request;

// 3. CLASE - Extiende del controlador base
class EventoController extends Controller
{
    /**
     * 4. MÉTODOS - Acciones del controlador
     * Cada método maneja una petición HTTP
     */
    public function index()
    {
        // Lógica para listar eventos
        $eventos = Evento::all();
        return response()->json(['data' => $eventos]);
    }

    public function store(Request $request)
    {
        // Lógica para crear un evento
        $evento = Evento::create($request->all());
        return response()->json(['data' => $evento], 201);
    }
}

Imports necesarios

1. Controller (base)

use App\Http\Controllers\Controller;

¿Qué es? La clase base de todos los controladores.
¿Para qué? Proporciona funcionalidad común (middleware, validación).

2. Request

use Illuminate\Http\Request;

¿Qué es? Objeto que contiene los datos de la petición HTTP.
¿Para qué? Acceder a datos del formulario, headers, archivos, etc.

3. Modelos

use App\Models\Evento;
use App\Models\Sector;

¿Qué es? Los modelos Eloquent que vas a usar.
¿Para qué? Interactuar con la base de datos.

4. Facades (opcionales)

use Illuminate\Support\Facades\DB;
use Illuminate\Support\Facades\Hash;
use Illuminate\Support\Facades\Auth;

¿Qué son? Interfaces estáticas para servicios de Laravel.
¿Para qué? Transacciones, hashing, autenticación, etc.

Métodos estándar de un controlador REST

Laravel sigue la convención REST para nombrar métodos:

Método Verbo HTTP Ruta Propósito
index() GET /recursos Listar todos
store() POST /recursos Crear nuevo
show($id) GET /recursos/{id} Ver uno
update($id) PUT/PATCH /recursos/{id} Actualizar
destroy($id) DELETE /recursos/{id} Eliminar

Métodos adicionales comunes

  • create() - Mostrar formulario de creación (para vistas)
  • edit($id) - Mostrar formulario de edición (para vistas)

Nota: En APIs REST puras, solo usamos los 5 primeros.

Glosario de métodos y conceptos

index() - Listar recursos

public function index()
{
    $eventos = Evento::all();
    return response()->json(['data' => $eventos]);
}

Equivalente SQL: SELECT * FROM eventos
Cuándo usarlo: Para obtener una lista de recursos.

store() - Crear recurso

public function store(Request $request)
{
    $request->validate([
        'nombre' => 'required|string|max:255',
    ]);

    $evento = Evento::create($request->all());
    return response()->json(['data' => $evento], 201);
}

Equivalente SQL: INSERT INTO eventos ...
Cuándo usarlo: Para crear un nuevo recurso.

show($id) - Ver un recurso

public function show($id)
{
    $evento = Evento::findOrFail($id);
    return response()->json(['data' => $evento]);
}

Equivalente SQL: SELECT * FROM eventos WHERE id = ?
Cuándo usarlo: Para obtener los detalles de un recurso específico.

update($id) - Actualizar recurso

public function update(Request $request, $id)
{
    $evento = Evento::findOrFail($id);
    $evento->update($request->all());
    return response()->json(['data' => $evento]);
}

Equivalente SQL: UPDATE eventos SET ... WHERE id = ?
Cuándo usarlo: Para modificar un recurso existente.

Validación de datos

public function store(Request $request)
{
    // Validar datos
    $validated = $request->validate([
        'nombre' => 'required|string|max:255',
        'email' => 'required|email|unique:users',
        'password' => 'required|min:8|confirmed',
        'edad' => 'nullable|integer|min:18',
    ]);
    
    // Si la validación falla, Laravel devuelve error 422 automáticamente
    // Si pasa, continúa con el código
    
    $user = User::create($validated);
}

Middleware en controladores

Los middleware filtran peticiones antes de llegar al controlador.

Aplicar middleware

class EventoController extends Controller
{
    public function __construct()
    {
        // Aplicar a todos los métodos
        $this->middleware('auth');
        
        // Aplicar solo a algunos métodos
        $this->middleware('auth')->only(['store', 'update', 'destroy']);
        
        // Aplicar excepto a algunos
        $this->middleware('auth')->except(['index', 'show']);
    }
}

Transacciones de base de datos

Para operaciones que involucran múltiples tablas:

use Illuminate\Support\Facades\DB;

public function store(Request $request)
{
    DB::beginTransaction();
    try {
        // Operación 1
        $evento = Evento::create($request->evento);
        
        // Operación 2
        foreach ($request->precios as $precio) {
            Precio::create([
                'evento_id' => $evento->id,
                'sector_id' => $precio['sector_id'],
                'precio' => $precio['precio'],
            ]);
        }
        
        DB::commit();
        return response()->json(['data' => $evento], 201);
        
    } catch (\Exception $e) {
        DB::rollBack();
        return response()->json(['error' => $e->getMessage()], 500);
    }
}

public function index(Request $request)
{
    $query = Evento::query();

Errores comunes

Error: “Class not found”

Causa: Falta el import del modelo o clase.
Solución:

use App\Models\Evento;

Error: “Method not allowed”

Causa: Verbo HTTP incorrecto en la ruta.
Solución: Verificar que la ruta use el verbo correcto (GET, POST, PUT, DELETE).

Error: “Undefined variable”

Causa: Variable no existe en el scope.
Solución: Verificar que la variable esté definida antes de usarla.

Error: “Call to undefined method”

Causa: Método no existe en el modelo o controlador.
Solución: Verificar el nombre del método y que esté definido.
