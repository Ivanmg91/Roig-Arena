Guía de Creación de Services para Roig Arena

Esta guía explica cómo crear Services (capa de servicios) para encapsular la lógica de negocio compleja y mantener los controladores limpios.

¿Qué son los Services?

Los Services son clases PHP que encapsulan lógica de negocio compleja. Separan la lógica de los controladores, haciéndolos más delgados, reutilizables y fáciles de testear.

¿Por qué usar Services?

Sin Services (❌ Controlador gordo)

// ReservaController
public function store(Request $request)
{
    // Validación
    $request->validate([...]);
    
    // Verificar disponibilidad
    $asiento = Asiento::findOrFail($request->asiento_id);
    if (!$asiento->estaDisponibleParaEvento($request->evento_id)) {
        return response()->json(['error' => 'No disponible'], 400);
    }
    
    // Verificar sector
    $evento = Evento::findOrFail($request->evento_id);
    if (!$evento->sectorEstaDisponible($asiento->sector_id)) {
        return response()->json(['error' => 'Sector no disponible'], 400);
    }
    
    // Crear reserva
    DB::beginTransaction();
    try {
        $reserva = EstadoAsiento::create([...]);
        
        // Enviar email
        Mail::to($user)->send(new ReservaConfirmada($reserva));
        
        // Registrar en logs
        Log::info('Reserva creada', ['reserva_id' => $reserva->id]);
        
        DB::commit();
    } catch (\Exception $e) {
        DB::rollBack();
        throw $e;
    }
    
    return response()->json(['data' => $reserva], 201);
}
// Controlador con 50+ líneas de lógica

// ReservaController
public function store(Request $request, ReservaService $service)

Services del proyecto

1. ReservaService

Propósito: Gestionar el proceso de reserva de asientos (carrito).

Ubicación: app/Services/ReservaService.php

Responsabilidades:

  • Verificar disponibilidad del asiento (con protección contra race condition)
  • Verificar que el sector esté disponible para el evento
  • Crear la reserva temporal (15 minutos)
  • Manejar errores y transacciones

Protección contra race condition:

  • Usa lockForUpdate() para bloqueo pesimista
  • Evita que 2 usuarios reserven el mismo asiento simultáneamente
  • La verificación y creación son atómicas dentro de la transacción

Código completo:

<?php

namespace App\Services;

use App\Models\EstadoAsiento;
use App\Models\Asiento;
use App\Models\Evento;
use Illuminate\Support\Facades\DB;

class ReservaService
{
    /**
     * Reservar un asiento para un evento
     */
    public function reservarAsiento($eventoId, $asientoId, $userId)
    {
        DB::beginTransaction();
        try {
            // Bloqueo pesimista: evita race condition
            $existeReserva = EstadoAsiento::where('evento_id', $eventoId)
                ->where('asiento_id', $asientoId)
                ->lockForUpdate()
                ->first();
            
            if ($existeReserva) {
                throw new \Exception('El asiento no está disponible');
            }
            
            $asiento = Asiento::findOrFail($asientoId);
            $evento = Evento::findOrFail($eventoId);
            
            $this->verificarSectorDisponible($evento, $asiento->sector_id);
            
            $reserva = EstadoAsiento::create([
                'evento_id' => $eventoId,
                'asiento_id' => $asientoId,
                'user_id' => $userId,
                'estado' => 'bloqueado',
                'reservado_hasta' => now()->addMinutes(15),
            ]);

            DB::commit();
            
            return $reserva->load(['evento', 'asiento.sector']);
            
        } catch (\Exception $e) {
            DB::rollBack();
            throw $e;
        }
    }

    /**
     * Cancelar una reserva
     */
    public function cancelarReserva($reservaId, $userId)
    {
        $reserva = EstadoAsiento::where('id', $reservaId)
            ->where('user_id', $userId)
            ->where('estado', 'bloqueado')
            ->firstOrFail();

        $reserva->delete();
        
        return true;
    }

    /**
     * Obtener reservas activas de un usuario
     */
    public function obtenerReservasActivas($userId)
    {
        return EstadoAsiento::where('user_id', $userId)
            ->where('estado', 'bloqueado')
            ->where('reservado_hasta', '>', now())
            ->with(['evento', 'asiento.sector'])
            ->get();
    }

    /**
     * Verificar que el sector esté disponible para el evento
     */
    private function verificarSectorDisponible($evento, $sectorId)
    {
        if (!$evento->sectorEstaDisponible($sectorId)) {
            throw new \Exception('El sector no está disponible para este evento');
        }
    }
}

🔒 Protección contra Race Condition

El problema

Cuando 2 usuarios intentan reservar el mismo asiento simultáneamente:

Usuario A: Verifica disponibilidad → ✅ Libre
Usuario B: Verifica disponibilidad → ✅ Libre
Usuario A: Crea reserva → ✅ Éxito
Usuario B: Crea reserva → ✅ Éxito
Resultado: 2 reservas del mismo asiento ❌

La solución: lockForUpdate()

El método lockForUpdate() implementa un bloqueo pesimista que evita este problema:

DB::beginTransaction();
try {
    // Bloqueo pesimista: evita race condition
    $existeReserva = EstadoAsiento::where('evento_id', $eventoId)
        ->where('asiento_id', $asientoId)
        ->lockForUpdate()  // ← Bloquea la fila
        ->first();
    
    if ($existeReserva) {
        throw new \Exception('El asiento no está disponible');
    }
    
    // Crear reserva
    $reserva = EstadoAsiento::create([...]);
    
    DB::commit();  // Libera el bloqueo
}

Cómo funciona

Usuario A: Inicia transacción
Usuario A: lockForUpdate() → 🔒 BLOQUEA la fila
Usuario B: Inicia transacción
Usuario B: Intenta lockForUpdate() → ⏳ ESPERA (bloqueado por A)
Usuario A: Verifica → Libre
Usuario A: Crea reserva → ✅ Éxito
Usuario A: Commit → 🔓 DESBLOQUEA
Usuario B: Ahora puede leer → Reserva existe
Usuario B: Verifica → Ocupado
Usuario B: Lanza excepción → ❌ "El asiento no está disponible"
Usuario B: Rollback
Resultado: Solo 1 reserva ✅

SQL generado

-- lockForUpdate() genera:
SELECT * FROM estado_asientos 
WHERE evento_id = ? AND asiento_id = ? 
FOR UPDATE;

-- Esto bloquea la fila hasta que termine la transacción (COMMIT o ROLLBACK)

Ventajas

✅ Prevención: Evita el problema antes de que ocurra
✅ Atómico: Verificación y creación son una operación única
✅ Seguro: Garantiza consistencia de datos
✅ Estándar: Práctica recomendada en Laravel

Niveles de protección

  1. Base de datos (constraint):
// Migración
$table->unique(['evento_id', 'asiento_id']);
  • Última línea de defensa
  • Genera error si se intenta duplicar
  1. Aplicación (lockForUpdate):
->lockForUpdate()
  • Previene el problema
  • Manejo elegante del error

2. CompraService

Propósito: Gestionar el proceso completo de compra (reserva → entrada con QR).

Ubicación: app/Services/CompraService.php

Responsabilidades:

  • Validar que las reservas no hayan expirado
  • Obtener precios de los sectores
  • Cambiar estado de reservas a “vendido”
  • Crear entradas con código QR
  • Manejar transacciones DB

Código completo:

<?php

namespace App\Services;

use App\Models\EstadoAsiento;
use App\Models\Entrada;
use Illuminate\Support\Facades\DB;

class CompraService
{
    /**
     * Procesar compra de múltiples reservas
     */
    public function procesarCompra(array $reservasIds, $userId)
    {
        $entradas = [];

        DB::beginTransaction();
        try {
            foreach ($reservasIds as $reservaId) {
                $reserva = $this->obtenerReserva($reservaId, $userId);
                
                // Verificar expiración
                $this->verificarNoExpirada($reserva);
                
                // Obtener precio
                $precio = $this->obtenerPrecio($reserva);
                
                // Marcar como vendido
                $reserva->marcarComoVendido();
                
                // Crear entrada
                $entrada = $this->crearEntrada($reserva, $precio, $userId);
                
                $entradas[] = $entrada;
            }

            DB::commit();
            
            return collect($entradas)->load(['evento', 'asiento.sector']);
            
        } catch (\Exception $e) {
            DB::rollBack();
            throw $e;
        }
    }

    /**
     * Obtener una reserva del usuario
     */
    private function obtenerReserva($reservaId, $userId)
    {
        return EstadoAsiento::where('id', $reservaId)
            ->where('user_id', $userId)
            ->where('estado', 'bloqueado')
            ->with(['evento', 'asiento.sector'])
            ->firstOrFail();
    }

    /**
     * Verificar que la reserva no haya expirado
     */
    private function verificarNoExpirada($reserva)
    {
        if ($reserva->haExpirado()) {
            throw new \Exception('Una de las reservas ha expirado');
        }
    }

    /**
     * Obtener el precio del sector para el evento
     */
    private function obtenerPrecio($reserva)
    {
        $precio = $reserva->evento->precioDelSector($reserva->asiento->sector_id);
        
        if (!$precio) {
            throw new \Exception('No se encontró el precio para el sector');
        }
        
        return $precio;
    }

    /**
     * Crear la entrada
     */
    private function crearEntrada($reserva, $precio, $userId)
    {
        return Entrada::create([
            'user_id' => $userId,
            'evento_id' => $reserva->evento_id,
            'asiento_id' => $reserva->asiento_id,
            'precio_pagado' => $precio->precio,
        ]);
    }
}

Uso en controlador:

// CompraController
public function store(Request $request, CompraService $service)
{
    $request->validate([
        'reservas' => 'required|array|min:1',
        'reservas.*' => 'exists:estado_asientos,id',
    ]);

    try {
        $entradas = $service->procesarCompra(
            $request->reservas,
            auth()->id()
        );

        return response()->json([
            'data' => EntradaResource::collection($entradas),
            'message' => 'Compra realizada correctamente',
        ], 201);
        
    } catch (\Exception $e) {
        return response()->json([
            'error' => $e->getMessage(),
        ], 400);
    }
}

3. LiberarReservasService

Propósito: Liberar reservas expiradas (comando programado).

Ubicación: app/Services/LiberarReservasService.php

Responsabilidades:

  • Buscar reservas expiradas
  • Eliminarlas de la base de datos
  • Registrar en logs

Código completo:

<?php

namespace App\Services;

use App\Models\EstadoAsiento;
use Illuminate\Support\Facades\Log;

class LiberarReservasService
{
    /**
     * Liberar todas las reservas expiradas
     */
    public function liberarExpiradas()
    {
        $expiradas = EstadoAsiento::expirados()->get();
        
        $count = 0;
        
        foreach ($expiradas as $reserva) {
            $reserva->delete();
            $count++;
            
            Log::info('Reserva expirada liberada', [
                'reserva_id' => $reserva->id,
                'evento_id' => $reserva->evento_id,
                'asiento_id' => $reserva->asiento_id,
            ]);
        }
        
        return $count;
    }

    /**
     * Liberar reservas de un usuario específico
     */
    public function liberarDeUsuario($userId)
    {
        $expiradas = EstadoAsiento::expirados()
            ->where('user_id', $userId)
            ->get();
        
        foreach ($expiradas as $reserva) {
            $reserva->delete();
        }
        
        return $expiradas->count();
    }
}

Uso en comando programado:

// app/Console/Commands/LiberarReservasExpiradas.php
<?php

namespace App\Console\Commands;

use App\Services\LiberarReservasService;
use Illuminate\Console\Command;

class LiberarReservasExpiradas extends Command
{
    protected $signature = 'reservas:liberar';
    protected $description = 'Liberar reservas expiradas automáticamente';

    public function handle(LiberarReservasService $service)
    {
        $count = $service->liberarExpiradas();
        
        $this->info("Se liberaron {$count} reservas expiradas");
        
        return 0;
    }
}

Apéndice: Services en Laravel para Principiantes

Esta guía te ayudará a entender qué son los Services, cuándo usarlos y cómo implementarlos correctamente.

¿Qué son los Services?

Los Services (Servicios) son clases PHP que encapsulan lógica de negocio compleja. Actúan como una capa intermedia entre los controladores y los modelos.

Analogía simple

Imagina una agencia de viajes:

  • Cliente = Usuario de la API
  • Recepcionista = Controlador (recibe peticiones)
  • Agente de viajes = Service (organiza todo el viaje)
  • Proveedores = Modelos (hoteles, vuelos, etc.)

El recepcionista no organiza el viaje completo, solo toma la petición y la pasa al agente especializado. El agente coordina con múltiples proveedores para crear el paquete completo.

¿Por qué usar Services?

Problema: Controladores gordos

// ReservaController - 80 líneas de lógica
public function store(Request $request)
{
    // Validación
    $request->validate([...]);
    
    // Buscar modelos
    $asiento = Asiento::findOrFail($request->asiento_id);
    $evento = Evento::findOrFail($request->evento_id);
    $user = auth()->user();
    
    // Verificación 1
    if (!$asiento->estaDisponibleParaEvento($evento->id)) {
        return response()->json(['error' => 'No disponible'], 400);
    }
    
    // Verificación 2
    if (!$evento->sectorEstaDisponible($asiento->sector_id)) {
        return response()->json(['error' => 'Sector no disponible'], 400);
    }
    
    // Verificación 3
    $precio = $evento->precioDelSector($asiento->sector_id);
    if (!$precio || !$precio->disponible) {
        return response()->json(['error' => 'Sin precio'], 400);
    }
    
    // Transacción
    DB::beginTransaction();
    try {
        // Crear reserva
        $reserva = EstadoAsiento::create([
            'evento_id' => $evento->id,
            'asiento_id' => $asiento->id,
            'user_id' => $user->id,
            'estado' => 'bloqueado',
            'reservado_hasta' => now()->addMinutes(15),
        ]);
        
        // Enviar email
        Mail::to($user)->send(new ReservaConfirmada($reserva));
        
        // Log
        Log::info('Reserva creada', ['id' => $reserva->id]);
        
        DB::commit();
        
        return response()->json(['data' => $reserva], 201);
        
    } catch (\Exception $e) {
        DB::rollBack();
        return response()->json(['error' => $e->getMessage()], 500);
    }
}

Problemas:

  • ❌ Controlador con 80+ líneas
  • ❌ Difícil de leer y mantener
  • ❌ Difícil de testear
  • ❌ Lógica no reutilizable
  • ❌ Mezcla responsabilidades

Solución: Services

  • ✅ Fácil de leer
  • ✅ Fácil de testear
  • ✅ Lógica reutilizable
  • ✅ Responsabilidades separadas

Cuándo usar Services

✅ SÍ usar Services cuando:

  1. Operación involucra múltiples modelos
// Compra: EstadoAsiento + Entrada + Email + Log
public function procesarCompra($reservas, $userId) { }
  1. Lógica compleja con múltiples pasos
// 1. Verificar disponibilidad
// 2. Verificar sector
// 3. Crear reserva
// 4. Enviar email
// 5. Registrar log
  1. Necesitas transacciones DB
DB::beginTransaction();
// Múltiples operaciones que deben ser atómicas
DB::commit();
  1. Lógica que se repite en varios controladores
// Verificar disponibilidad se usa en:
// - ReservaController
// - CompraController
// - AsientoController
  1. Operaciones programadas (cron jobs)
// Liberar reservas expiradas cada minuto
$schedule->command('reservas:liberar')->everyMinute();

❌ NO usar Services cuando:

  1. CRUD simple de un solo modelo
// EventoController - No necesita Service
public function index()
{
    return EventoResource::collection(Evento::all());
}

public function store(Request $request)
{
    $evento = Evento::create($request->validated());
    return new EventoResource($evento);
}
  1. Operación de lectura simple
// No necesita Service
public function show($id)
{
    $evento = Evento::findOrFail($id);
    return new EventoResource($evento);
}
  1. Validación simple
// Usa Form Requests, no Services
$request->validate([...]);

Estructura de un Service

Ubicación

app/Services/
├── ReservaService.php
├── CompraService.php
└── LiberarReservasService.php

Anatomía básica

<?php

// 1. NAMESPACE
namespace App\Services;

// 2. IMPORTS
use App\Models\EstadoAsiento;
use App\Models\Asiento;
use Illuminate\Support\Facades\DB;

// 3. CLASE
class ReservaService
{
    /**
     * 4. MÉTODOS PÚBLICOS - Usados por controladores
     */
    public function reservarAsiento($eventoId, $asientoId, $userId)
    {
        // Lógica principal
        DB::beginTransaction();
        try {
            // ...
            DB::commit();
            return $resultado;
        } catch (\Exception $e) {
            DB::rollBack();
            throw $e;
        }
    }
    
    /**
     * 5. MÉTODOS PRIVADOS - Lógica auxiliar
     */
    private function verificarDisponibilidad($asiento, $eventoId)
    {
        if (!$asiento->estaDisponibleParaEvento($eventoId)) {
            throw new \Exception('No disponible');
        }
    }
}

Imports necesarios

Modelos

use App\Models\EstadoAsiento;
use App\Models\Asiento;
use App\Models\Evento;
use App\Models\Entrada;

Facades comunes

use Illuminate\Support\Facades\DB;      // Transacciones
use Illuminate\Support\Facades\Log;     // Logs
use Illuminate\Support\Facades\Mail;    // Emails
use Illuminate\Support\Facades\Cache;   // Caché

Inyección de dependencias

Laravel inyecta automáticamente los Services en los controladores:

// Controlador
public function store(Request $request, ReservaService $service)
{
    // $service se inyecta automáticamente
    $reserva = $service->reservarAsiento(...);
}

No necesitas hacer:

// ❌ No hagas esto
$service = new ReservaService();
$service = app(ReservaService::class);

// ✅ Deja que Laravel lo inyecte
public function store(Request $request, ReservaService $service)

Transacciones de base de datos

Las transacciones aseguran que todas las operaciones se completen o ninguna:

DB::beginTransaction();
try {
    // Operación 1
    $reserva = EstadoAsiento::create([...]);
    
    // Operación 2
    Mail::to($user)->send(new Email());
    
    // Operación 3
    Log::info('Reserva creada');
    
    // Si todo va bien, confirmar
    DB::commit();
    
    return $reserva;
    
} catch (\Exception $e) {
    // Si algo falla, revertir TODO
    DB::rollBack();
    
    // Re-lanzar la excepción
    throw $e;
}

¿Por qué? Si falla el email, la reserva no debe crearse. Todo o nada.

Manejo de errores

Lanzar excepciones

private function verificarDisponibilidad($asiento, $eventoId)
{
    if (!$asiento->estaDisponibleParaEvento($eventoId)) {
        throw new \Exception('El asiento no está disponible');
    }
}

Capturar en el controlador

public function store(Request $request, ReservaService $service)
{
    try {
        $reserva = $service->reservarAsiento(...);
        return response()->json(['data' => $reserva], 201);
        
    } catch (\Exception $e) {
        return response()->json([
            'error' => $e->getMessage(),
        ], 400);
    }
}

Métodos públicos vs privados

Métodos públicos

  • Usados por controladores
  • Representan acciones principales
  • Nombres descriptivos
public function reservarAsiento($eventoId, $asientoId, $userId)
{
    // Acción principal
}

public function cancelarReserva($reservaId, $userId)
{
    // Acción principal
}

Métodos privados

  • Usados internamente por el Service
  • Lógica auxiliar
  • Validaciones, cálculos, etc.
private function verificarDisponibilidad($asiento, $eventoId)
{
    // Lógica auxiliar
}

private function obtenerPrecio($evento, $sectorId)
{
    // Lógica auxiliar
}

Patrones comunes

Patrón 1: Verificar y ejecutar

public function reservarAsiento($eventoId, $asientoId, $userId)
{
    // 1. Obtener datos
    $asiento = Asiento::findOrFail($asientoId);
    $evento = Evento::findOrFail($eventoId);
    
    // 2. Verificar condiciones
    $this->verificarDisponibilidad($asiento, $eventoId);
    $this->verificarSector($evento, $asiento->sector_id);
    
    // 3. Ejecutar operación
    return $this->crearReserva($eventoId, $asientoId, $userId);
}

Patrón 2: Transacción con múltiples operaciones

public function procesarCompra($reservasIds, $userId)
{
    DB::beginTransaction();
    try {
        $entradas = [];
        
        foreach ($reservasIds as $reservaId) {
            $entrada = $this->procesarReserva($reservaId, $userId);
            $entradas[] = $entrada;
        }
        
        DB::commit();
        return $entradas;
        
    } catch (\Exception $e) {
        DB::rollBack();
        throw $e;
    }
}

Patrón 3: Operación con logging

public function liberarReservasExpiradas()
{
    $expiradas = EstadoAsiento::expirados()->get();
    
    foreach ($expiradas as $reserva) {
        $reserva->delete();
        
        Log::info('Reserva liberada', [
            'reserva_id' => $reserva->id,
            'evento_id' => $reserva->evento_id,
        ]);
    }
    
    return $expiradas->count();
}

Retornar datos

Retornar modelos cargados

// ✅ Bien - Retorna con relaciones cargadas
return $reserva->load(['evento', 'asiento.sector']);

// ❌ Mal - Sin relaciones
return $reserva;

Retornar colecciones

// ✅ Bien - Retorna colección de Laravel
return collect($entradas)->load(['evento', 'asiento']);

// ✅ También bien - Array de modelos
return $entradas;

No formatear en Services

// ❌ Mal - Formateo en Service
return [
    'id' => $reserva->id,
    'fecha' => $reserva->created_at->format('d/m/Y'),
];

// ✅ Bien - Retorna el modelo, formateo en Resource
return $reserva;

Testing de Services

Los Services son fáciles de testear porque no dependen de HTTP:

// tests/Unit/ReservaServiceTest.php
<?php

namespace Tests\Unit;

use Tests\TestCase;
use App\Services\ReservaService;
use App\Models\Evento;
use App\Models\Asiento;
use App\Models\User;

class ReservaServiceTest extends TestCase
{
    public function test_puede_reservar_asiento_disponible()
    {
        // Arrange
        $evento = Evento::factory()->create();
        $asiento = Asiento::factory()->create();
        $user = User::factory()->create();
        
        $service = new ReservaService();
        
        // Act
        $reserva = $service->reservarAsiento(
            $evento->id,
            $asiento->id,
            $user->id
        );
        
        // Assert
        $this->assertNotNull($reserva);
        $this->assertEquals('bloqueado', $reserva->estado);
        $this->assertEquals($evento->id, $reserva->evento_id);
    }
    
    public function test_no_puede_reservar_asiento_ocupado()
    {
        // Arrange
        $evento = Evento::factory()->create();
        $asiento = Asiento::factory()->create();
        $user = User::factory()->create();
        
        // Crear reserva existente
        EstadoAsiento::create([
            'evento_id' => $evento->id,
            'asiento_id' => $asiento->id,
            'user_id' => $user->id,
            'estado' => 'bloqueado',
            'reservado_hasta' => now()->addMinutes(15),
        ]);
        
        $service = new ReservaService();
        
        // Act & Assert
        $this->expectException(\Exception::class);
        $this->expectExceptionMessage('no está disponible');
        
        $service->reservarAsiento($evento->id, $asiento->id, $user->id);
    }
}

// CompraService - Solo compras

DB::beginTransaction();
try {

// Lanzar excepciones descriptivas

Errores comunes

Error: “Class not found”

Causa: Falta el namespace o import.
Solución:

namespace App\Services;
use App\Models\EstadoAsiento;

Error: “Too few arguments”

Causa: Faltan parámetros al llamar al método.
Solución: Verifica que pasas todos los parámetros requeridos.

Error: “Call to undefined method”

Causa: Método no existe o es privado.
Solución: Verifica el nombre y que sea público si lo llamas desde fuera.

Resumen rápido (Cheat Sheet)

Estructura básica

namespace App\Services;

use App\Models\Modelo;
use Illuminate\Support\Facades\DB;

class MiService
{
    public function operacionPrincipal($param1, $param2)
    {
        DB::beginTransaction();
        try {
            // Lógica
            DB::commit();
            return $resultado;
        } catch (\Exception $e) {
            DB::rollBack();
            throw $e;
        }
    }
    
    private function auxiliar($param)
    {
        // Lógica auxiliar
    }
}

Uso en controlador

public function store(Request $request, MiService $service)
{
    try {
        $resultado = $service->operacionPrincipal(...);
        return response()->json(['data' => $resultado], 201);
    } catch (\Exception $e) {
        return response()->json(['error' => $e->getMessage()], 400);
    }
}

Transacciones

DB::beginTransaction();
try {
    // Operaciones
    DB::commit();
} catch (\Exception $e) {
    DB::rollBack();
    throw $e;
}
