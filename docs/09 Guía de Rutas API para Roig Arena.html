Guía de Rutas API para Roig Arena

Esta guía explica cómo definir todas las rutas de la API REST del sistema de venta de entradas.

¿Qué son las rutas?

Las rutas son mapeos entre URLs y controladores. Definen qué controlador y método se ejecuta cuando se hace una petición HTTP a una URL específica.

Estructura de rutas del proyecto

Tipos de rutas

  1. Rutas públicas - Sin autenticación
  2. Rutas protegidas - Requieren autenticación (auth:sanctum)
  3. Rutas de administrador - Requieren autenticación + permisos (auth:sanctum + admin)

Archivo routes/api.php completo

Ubicación: routes/api.php

<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\Auth\AuthController;
use App\Http\Controllers\EventoController;
use App\Http\Controllers\SectorController;
use App\Http\Controllers\AsientoController;
use App\Http\Controllers\ReservaController;
use App\Http\Controllers\CompraController;
use App\Http\Controllers\EntradaController;

/*
|--------------------------------------------------------------------------
| API Routes
|--------------------------------------------------------------------------
|
| Rutas de la API REST para el sistema de venta de entradas Roig Arena
|
*/

// ============================================
// RUTAS PÚBLICAS (sin autenticación)
// ============================================

// Autenticación
Route::post('/register', [AuthController::class, 'register']);
Route::post('/login', [AuthController::class, 'login']);

// Eventos (consulta pública)
Route::get('/eventos', [EventoController::class, 'index']);
Route::get('/eventos/{id}', [EventoController::class, 'show']);

// Sectores (consulta pública)
Route::get('/sectores', [SectorController::class, 'index']);

// Asientos (consulta pública)
Route::get('/eventos/{eventoId}/asientos', [AsientoController::class, 'porEvento']);
Route::get('/eventos/{eventoId}/sectores/{sectorId}/asientos', [AsientoController::class, 'porSector']);

// ============================================
// RUTAS PROTEGIDAS (requieren autenticación)
// ============================================

Route::middleware('auth:sanctum')->group(function () {
    
    // Usuario autenticado
    Route::get('/user', [AuthController::class, 'user']);
    Route::post('/logout', [AuthController::class, 'logout']);
    
    // Reservas (carrito de compras)
    Route::get('/reservas', [ReservaController::class, 'index']);
    Route::post('/reservas', [ReservaController::class, 'store']);
    Route::delete('/reservas/{id}', [ReservaController::class, 'destroy']);
    
    // Compras
    Route::post('/compras', [CompraController::class, 'store']);
    
    // Entradas
    Route::get('/entradas', [EntradaController::class, 'index']);
    Route::get('/entradas/{id}', [EntradaController::class, 'show']);
});

// ============================================
// RUTAS DE ADMINISTRADOR
// ============================================

Route::middleware(['auth:sanctum', 'admin'])->prefix('admin')->group(function () {
    
    // Eventos (CRUD completo)
    Route::post('/eventos', [EventoController::class, 'store']);
    Route::put('/eventos/{id}', [EventoController::class, 'update']);
    Route::delete('/eventos/{id}', [EventoController::class, 'destroy']);
    
    // Sectores (CRUD completo)
    Route::post('/sectores', [SectorController::class, 'store']);
    Route::put('/sectores/{id}', [SectorController::class, 'update']);
    Route::delete('/sectores/{id}', [SectorController::class, 'destroy']);
});

Desglose de rutas

1. Rutas de Autenticación

// Registro de usuario
POST /api/register
Body: { nombre, apellido, email, password, password_confirmation }
Response: { user, token, token_type }

// Iniciar sesión
POST /api/login
Body: { email, password }
Response: { user, token, token_type }

// Cerrar sesión (protegida)
POST /api/logout
Headers: Authorization: Bearer {token}
Response: { message }

// Obtener usuario autenticado (protegida)
GET /api/user
Headers: Authorization: Bearer {token}
Response: { user }

2. Rutas de Eventos

// Listar eventos futuros (público)
GET /api/eventos
Response: { data: [eventos] }

// Ver detalle de evento (público)
GET /api/eventos/{id}
Response: { data: evento }

// Crear evento (admin)
POST /api/admin/eventos
Headers: Authorization: Bearer {token}
Body: { nombre, descripcion_corta, descripcion_larga, poster_url, fecha, hora }
Response: { data: evento, message }

// Actualizar evento (admin)
PUT /api/admin/eventos/{id}
Headers: Authorization: Bearer {token}
Body: { nombre?, descripcion_corta?, ... }
Response: { data: evento, message }

// Eliminar evento (admin)
DELETE /api/admin/eventos/{id}
Headers: Authorization: Bearer {token}
Response: { message }

3. Rutas de Sectores

// Listar sectores activos (público)
GET /api/sectores
Response: { data: [sectores] }

// Crear sector (admin)
POST /api/admin/sectores
Headers: Authorization: Bearer {token}
Body: { nombre, descripcion?, activo? }
Response: { data: sector, message }

// Actualizar sector (admin)
PUT /api/admin/sectores/{id}
Headers: Authorization: Bearer {token}
Body: { nombre?, descripcion?, activo? }
Response: { data: sector, message }

// Eliminar sector (admin)
DELETE /api/admin/sectores/{id}
Headers: Authorization: Bearer {token}
Response: { message }

4. Rutas de Asientos

// Asientos disponibles de un evento (público)
GET /api/eventos/{eventoId}/asientos
Response: { data: [asientos] }

// Asientos de un sector específico (público)
GET /api/eventos/{eventoId}/sectores/{sectorId}/asientos
Response: { data: { sector, precio, asientos } }

5. Rutas de Reservas (Carrito)

// Ver mis reservas activas (protegida)
GET /api/reservas
Headers: Authorization: Bearer {token}
Response: { data: [reservas] }

// Reservar un asiento (protegida)
POST /api/reservas
Headers: Authorization: Bearer {token}
Body: { evento_id, asiento_id }
Response: { data: reserva, message }

// Cancelar reserva (protegida)
DELETE /api/reservas/{id}
Headers: Authorization: Bearer {token}
Response: { message }

6. Rutas de Compras

// Confirmar compra de reservas (protegida)
POST /api/compras
Headers: Authorization: Bearer {token}
Body: { reservas: [id1, id2, ...] }
Response: { data: [entradas], message }

7. Rutas de Entradas

// Listar mis entradas (protegida)
GET /api/entradas
Headers: Authorization: Bearer {token}
Response: { data: [entradas] }

// Ver detalle de entrada (protegida)
GET /api/entradas/{id}
Headers: Authorization: Bearer {token}
Response: { data: entrada }

Prefijos y agrupación

Prefijo /api

Todas las rutas API tienen el prefijo /api automáticamente:

  • Definido en bootstrap/app.php
  • No necesitas escribirlo en routes/api.php
// En routes/api.php escribes:
Route::get('/eventos', [EventoController::class, 'index']);

// La URL real es:
GET http://localhost/api/eventos

Agrupación por middleware

// Grupo con un middleware
Route::middleware('auth:sanctum')->group(function () {
    // Todas estas rutas requieren autenticación
});

// Grupo con múltiples middleware
Route::middleware(['auth:sanctum', 'admin'])->group(function () {
    // Todas estas rutas requieren autenticación + admin
});

// Grupo con prefijo
Route::prefix('admin')->group(function () {
    // Todas estas rutas tienen /admin/ al inicio
});

// Combinado
Route::middleware(['auth:sanctum', 'admin'])
    ->prefix('admin')
    ->group(function () {
        // /api/admin/* con autenticación + admin
    });

Verbos HTTP

Verbo Propósito Ejemplo
GET Obtener datos Listar eventos
POST Crear recurso Crear evento
PUT Actualizar completo Actualizar evento
PATCH Actualizar parcial Actualizar solo nombre
DELETE Eliminar recurso Eliminar evento 
Route::get('/eventos', [EventoController::class, 'index']);      // Listar
Route::post('/eventos', [EventoController::class, 'store']);     // Crear
Route::get('/eventos/{id}', [EventoController::class, 'show']);  // Ver uno
Route::put('/eventos/{id}', [EventoController::class, 'update']); // Actualizar
Route::delete('/eventos/{id}', [EventoController::class, 'destroy']); // Eliminar

Parámetros de ruta

Parámetros obligatorios

Route::get('/eventos/{id}', [EventoController::class, 'show']);
// URL: /api/eventos/1
// Controlador recibe: $id = 1

Route::get('/eventos/{eventoId}/sectores/{sectorId}', [AsientoController::class, 'porSector']);
// URL: /api/eventos/1/sectores/2
// Controlador recibe: $eventoId = 1, $sectorId = 2

Parámetros opcionales

Route::get('/eventos/{id?}', [EventoController::class, 'show']);
// URL: /api/eventos o /api/eventos/1
// Controlador recibe: $id = null o $id = 1

Restricciones

// Solo números
Route::get('/eventos/{id}', [EventoController::class, 'show'])
    ->where('id', '[0-9]+');

// Solo letras
Route::get('/categorias/{slug}', [CategoriaController::class, 'show'])
    ->where('slug', '[a-z-]+');

Resource Routes (opcional)

Laravel ofrece un atajo para rutas CRUD:

// En lugar de escribir todas las rutas manualmente:
Route::get('/eventos', [EventoController::class, 'index']);
Route::post('/eventos', [EventoController::class, 'store']);
Route::get('/eventos/{id}', [EventoController::class, 'show']);
Route::put('/eventos/{id}', [EventoController::class, 'update']);
Route::delete('/eventos/{id}', [EventoController::class, 'destroy']);

// Puedes usar:
Route::apiResource('eventos', EventoController::class);

Nota: Para este proyecto usamos rutas explícitas para mayor claridad.

Throttling (Limitar peticiones)

Proteger contra ataques:

// Limitar login a 5 intentos por minuto
Route::middleware('throttle:5,1')->group(function () {
    Route::post('/login', [AuthController::class, 'login']);

});

// Limitar API general a 60 peticiones por minuto
Route::middleware(['auth:sanctum', 'throttle:60,1'])->group(function () {
    // Rutas protegidas
});

Apéndice: Rutas en Laravel para Principiantes

Esta guía te ayudará a entender qué son las rutas, cómo funcionan y cómo definirlas correctamente.

¿Qué son las rutas?

Las rutas son mapeos entre URLs y código. Definen qué sucede cuando alguien visita una URL específica en tu aplicación.

Analogía simple

Imagina un edificio de oficinas:

  • Dirección del edificio = Dominio (example.com)
  • Número de oficina = Ruta (/api/eventos)
  • Recepcionista = Router (Laravel)
  • Empleado = Controlador

Cuando llegas al edificio (dominio) y dices “busco la oficina 305” (ruta), el recepcionista (router) te dirige al empleado correcto (controlador).

¿Por qué usar rutas?

Sin rutas organizadas

// Todo mezclado en un solo archivo
if ($_SERVER['REQUEST_URI'] == '/eventos' && $_SERVER['REQUEST_METHOD'] == 'GET') {
    // Código para listar eventos
} elseif ($_SERVER['REQUEST_URI'] == '/eventos' && $_SERVER['REQUEST_METHOD'] == 'POST') {
    // Código para crear evento
}
// Difícil de mantener ❌

Con rutas de Laravel

// Organizado y claro
Route::get('/eventos', [EventoController::class, 'index']);
Route::post('/eventos', [EventoController::class, 'store']);
// Fácil de mantener ✅

Ubicación de archivos de rutas

routes/
├── web.php      # Rutas web (con sesiones, CSRF)
├── api.php      # Rutas API (sin CSRF, con Sanctum)
└── console.php  # Comandos de consola

Diferencias

Archivo Prefijo Middleware Uso
web.php / web, csrf Aplicaciones web tradicionales
api.php /api api APIs REST

Estructura básica de una ruta

Route::verbo('/url', [Controlador::class, 'metodo']);

Componentes:

  1. Route:: - Clase de Laravel para definir rutas
  2. verbo - Método HTTP (get, post, put, delete)
  3. '/url' - La URL (sin el prefijo /api)
  4. [Controlador::class, 'metodo'] - Qué ejecutar

Ejemplo

Route::get('/eventos', [EventoController::class, 'index']);

Significado:

  • Cuando alguien hace GET a /api/eventos
  • Ejecuta el método index() del EventoController

Verbos HTTP

Los verbos HTTP indican la acción a realizar:

Verbo Propósito Ejemplo
GET Obtener/Leer Listar eventos
POST Crear Crear evento
PUT Actualizar completo Actualizar todos los campos
PATCH Actualizar parcial Actualizar solo algunos campos
DELETE Eliminar Eliminar evento

Ejemplos

// GET - Obtener datos
Route::get('/eventos', [EventoController::class, 'index']);
Route::get('/eventos/{id}', [EventoController::class, 'show']);

// POST - Crear
Route::post('/eventos', [EventoController::class, 'store']);

// PUT - Actualizar
Route::put('/eventos/{id}', [EventoController::class, 'update']);

// DELETE - Eliminar
Route::delete('/eventos/{id}', [EventoController::class, 'destroy']);

Parámetros de ruta

Parámetros obligatorios

Route::get('/eventos/{id}', [EventoController::class, 'show']);

URL: /api/eventos/1
Controlador recibe: $id = 1

public function show($id)
{
    $evento = Evento::findOrFail($id);
    return new EventoResource($evento);
}

Múltiples parámetros

Route::get('/eventos/{eventoId}/sectores/{sectorId}', [AsientoController::class, 'porSector']);

URL: /api/eventos/1/sectores/2
Controlador recibe: $eventoId = 1, $sectorId = 2

public function porSector($eventoId, $sectorId)
{
    // Usar ambos parámetros
}

Parámetros opcionales

Route::get('/eventos/{id?}', [EventoController::class, 'show']);

URLs válidas:

  • /api/eventos → $id = null
  • /api/eventos/1 → $id = 1
public function show($id = null)
{
    if ($id) {
        return Evento::findOrFail($id);
    }
    return Evento::all();
}

Restricciones de parámetros

// Solo números
Route::get('/eventos/{id}', [EventoController::class, 'show'])
    ->where('id', '[0-9]+');

// Solo letras minúsculas y guiones
Route::get('/categorias/{slug}', [CategoriaController::class, 'show'])
    ->where('slug', '[a-z-]+');

// Múltiples restricciones
Route::get('/eventos/{id}/comentarios/{comentarioId}', ...)
    ->where(['id' => '[0-9]+', 'comentarioId' => '[0-9]+']);

Middleware en rutas

Los middleware filtran las peticiones antes de llegar al controlador.

Aplicar a una ruta

Route::get('/admin/dashboard', [AdminController::class, 'index'])
    ->middleware('admin');

Múltiples middleware

Route::get('/admin/dashboard', [AdminController::class, 'index'])
    ->middleware(['auth:sanctum', 'admin']);

Grupos con middleware

Route::middleware('auth:sanctum')->group(function () {
    Route::get('/reservas', [ReservaController::class, 'index']);
    Route::post('/reservas', [ReservaController::class, 'store']);
    Route::delete('/reservas/{id}', [ReservaController::class, 'destroy']);
});

Grupos de rutas

Agrupar rutas relacionadas para aplicar configuración común.

Grupo con middleware

Route::middleware('auth:sanctum')->group(function () {
    // Todas estas rutas requieren autenticación
    Route::get('/user', [AuthController::class, 'user']);
    Route::post('/logout', [AuthController::class, 'logout']);
});

Grupo con prefijo

Route::prefix('admin')->group(function () {
    // /api/admin/eventos
    Route::get('/eventos', [EventoController::class, 'index']);
    // /api/admin/sectores
    Route::get('/sectores', [SectorController::class, 'index']);
});

Grupo combinado

Route::middleware(['auth:sanctum', 'admin'])
    ->prefix('admin')
    ->group(function () {
        // /api/admin/* con autenticación + admin
        Route::post('/eventos', [EventoController::class, 'store']);
        Route::put('/eventos/{id}', [EventoController::class, 'update']);
    });

Prefijo /api automático

En routes/api.php, todas las rutas tienen el prefijo /api automáticamente.

// En routes/api.php escribes:
Route::get('/eventos', [EventoController::class, 'index']);

// La URL real es:
GET http://localhost/api/eventos

Configurado en: bootstrap/app.php

->withRouting(
    api: __DIR__.'/../routes/api.php',  // Prefijo /api automático
)

Nombrar rutas

Puedes asignar nombres a las rutas para referenciarlas fácilmente.

Route::get('/eventos', [EventoController::class, 'index'])
    ->name('eventos.index');

Route::get('/eventos/{id}', [EventoController::class, 'show'])
    ->name('eventos.show');

Uso:

// Generar URL
$url = route('eventos.show', ['id' => 1]);
// Resultado: http://localhost/api/eventos/1

// Redireccionar
return redirect()->route('eventos.index');

// En vistas Blade
<a href="{{ route('eventos.show', ['id' => $evento->id]) }}">Ver evento</a>

Resource Routes

Laravel ofrece un atajo para rutas CRUD estándar.

apiResource

Route::apiResource('eventos', EventoController::class);

Genera automáticamente:

Verbo URL Método Nombre
GET /eventos index eventos.index
POST /eventos store eventos.store
GET /eventos/{id} show eventos.show
PUT /eventos/{id} update eventos.update
DELETE /eventos/{id} destroy eventos.destroy

Limitar métodos

// Solo algunos métodos
Route::apiResource('eventos', EventoController::class)
    ->only(['index', 'show']);

// Todos excepto algunos
Route::apiResource('eventos', EventoController::class)
    ->except(['destroy']);

Throttling (Limitar peticiones)

Proteger contra ataques de fuerza bruta.

// Limitar a 5 peticiones por minuto
Route::middleware('throttle:5,1')->group(function () {
    Route::post('/login', [AuthController::class, 'login']);
});

// Limitar a 60 peticiones por minuto
Route::middleware('throttle:60,1')->group(function () {
    Route::get('/eventos', [EventoController::class, 'index']);
});

Formato: throttle:intentos,minutos

Orden de definición

El orden de las rutas importa. Laravel usa la primera coincidencia.

// ❌ Mal - La segunda nunca se ejecuta
Route::get('/eventos/{id}', [EventoController::class, 'show']);
Route::get('/eventos/destacados', [EventoController::class, 'destacados']);
// /eventos/destacados coincide con /eventos/{id} donde id="destacados"

// ✅ Bien - Específicas primero
Route::get('/eventos/destacados', [EventoController::class, 'destacados']);
Route::get('/eventos/{id}', [EventoController::class, 'show']);

Respuestas desde rutas (Closures)

Puedes definir lógica directamente en la ruta (no recomendado para APIs).

// Respuesta simple
Route::get('/ping', function () {
    return response()->json(['message' => 'pong']);
});

// Con parámetros
Route::get('/saludo/{nombre}', function ($nombre) {
    return response()->json(['message' => "Hola, {$nombre}"]);
});

Nota: Para APIs, es mejor usar controladores.

Redirecciones

// Redirección simple
Route::redirect('/aqui', '/alla');

// Redirección permanente (301)
Route::permanentRedirect('/viejo', '/nuevo');

Fallback (Ruta 404)

Definir qué hacer cuando no se encuentra una ruta.

Route::fallback(function () {
    return response()->json([
        'error' => 'Ruta no encontrada',
    ], 404);
});

❌ Evitar

// No mezclar verbos
Route::get('/eventos/crear', ...); // ❌ Usar POST /eventos

// No usar verbos en URLs
Route::post('/crear-evento', ...); // ❌ Usar POST /eventos

// No URLs en singular
Route::get('/evento', ...); // ❌ Usar /eventos

// No lógica compleja en closures
Route::get('/eventos', function () {
    // 50 líneas de código ❌
});

// No rutas sin middleware cuando se necesita
Route::post('/compras', ...); // ❌ Falta auth:sanctum

Convenciones REST

Acción Verbo URL Controlador
Listar todos GET /recursos index()
Ver uno GET /recursos/{id} show($id)
Crear POST /recursos store()
Actualizar PUT /recursos/{id} update($id)
Eliminar DELETE /recursos/{id} destroy($id)

Errores comunes

Error: “Route not found”

Causa: La ruta no está definida o la URL es incorrecta.
Solución: Verificar con sail artisan route:list.

Error: “Method not allowed”

Causa: Verbo HTTP incorrecto.
Solución: Verificar que uses GET, POST, PUT o DELETE correctamente.

Error: “Controller not found”

Causa: Falta el import del controlador.
Solución:

use App\Http\Controllers\EventoController;

Error: “Too many requests”

Causa: Throttling excedido.
Solución: Esperar o ajustar el límite.

Resumen rápido (Cheat Sheet)

Definir rutas

Route::get('/url', [Controller::class, 'metodo']);
Route::post('/url', [Controller::class, 'metodo']);
Route::put('/url/{id}', [Controller::class, 'metodo']);
Route::delete('/url/{id}', [Controller::class, 'metodo']);

Con middleware

Route::middleware('auth:sanctum')->group(function () {
    // Rutas protegidas
});

Con prefijo

Route::prefix('admin')->group(function () {
    // /api/admin/*
});

Parámetros

Route::get('/eventos/{id}', ...);           // Obligatorio
Route::get('/eventos/{id?}', ...);          // Opcional
Route::get('/eventos/{id}', ...)->where('id', '[0-9]+'); // Restricción

Listar rutas

sail artisan route:list
sail artisan route:list --path=api
