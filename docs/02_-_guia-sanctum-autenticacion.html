Guía de Instalación y Configuración de Laravel Sanctum

Laravel Sanctum es el sistema oficial de autenticación de Laravel para APIs. Te permite crear tokens de acceso para autenticar peticiones desde aplicaciones móviles, SPAs (Single Page Applications) o APIs externas.

¿Qué es Laravel Sanctum?

Sanctum proporciona dos tipos de autenticación:

  1. Autenticación basada en tokens - Para aplicaciones móviles o APIs de terceros
  2. Autenticación basada en sesiones - Para SPAs (Vue, React, etc.) en el mismo dominio

En tu proyecto Roig Arena, lo usarás principalmente para:

  • Autenticar usuarios desde el frontend
  • Proteger rutas de API
  • Gestionar tokens de acceso
  • Diferenciar entre usuarios normales y administradores

Requisitos previos

  • Laravel Sail corriendo: ./vendor/bin/sail up -d
  • Estar en el directorio del proyecto: cd /arena

Paso 1: Verificar que Sanctum está instalado

Laravel 11+ incluye Sanctum por defecto. Verifica que está en tu composer.json:

./vendor/bin/sail composer show laravel/sanctum

Salida esperada:

name     : laravel/sanctum
descrip. : Laravel Sanctum provides a featherweight authentication system for SPAs and simple APIs.
versions : * v4.x.x

Si no está instalado (poco probable), instálalo:

./vendor/bin/sail composer require laravel/sanctum

Paso 2: Publicar la configuración de Sanctum

Este comando crea el archivo de configuración y las migraciones necesarias:

./vendor/bin/sail artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"

Paso 3: Configurar el modelo User

Tu modelo User debe usar el trait HasApiTokens de Sanctum.

3.1. Editar el modelo User

Abre el archivo app/Models/User.php y asegúrate de que tenga esto:

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\SoftDeletes;
use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;
use Laravel\Sanctum\HasApiTokens;

class User extends Authenticatable
{
    use HasFactory, Notifiable, HasApiTokens, SoftDeletes;

    /**
     * The attributes that are mass assignable.
     *
     * @var array<int, string>
     */
    protected $fillable = [
        'nombre',
        'apellido',
        'email',
        'password',
        'is_admin',
    ];

    /**
     * The attributes that should be hidden for serialization.
     *
     * @var array<int, string>
     */
    protected $hidden = [
        'password',
        'remember_token',
    ];

    /**
     * Get the attributes that should be cast.
     *
     * @return array<string, string>
     */
    protected function casts(): array
    {
        return [
            'email_verified_at' => 'datetime',
            'password' => 'hashed',
            'is_admin' => 'boolean',
        ];
    }

    /**
     * Verificar si el usuario es administrador
     */
    public function isAdmin(): bool
    {
        return $this->is_admin;
    }
}

Cambios importantes:

  • use Laravel\Sanctum\HasApiTokens; - Importar el trait
  • use HasApiTokens - Añadir el trait a la clase
  • use SoftDeletes - Para borrado lógico
  • fillable actualizado con tus campos (nombre, apellido, is_admin)
  • casts() incluye is_admin como boolean
  • Método isAdmin() para verificar permisos

Paso 4: Configurar CORS (Cross-Origin Resource Sharing)

Si tu frontend está en un dominio diferente, necesitas configurar CORS.

4.1. Editar config/cors.php

Abre config/cors.php y configúralo así:

<?php

return [
    'paths' => ['api/*', 'sanctum/csrf-cookie'],
    'allowed_methods' => ['*'],
    'allowed_origins' => ['*'], // En producción, especifica tu dominio
    'allowed_origins_patterns' => [],
    'allowed_headers' => ['*'],
    'exposed_headers' => [],
    'max_age' => 0,
    'supports_credentials' => true,
];

Para producción, cambia 'allowed_origins' => ['*'] por tu dominio:

'allowed_origins' => ['https://tudominio.com'],

        //

Paso 8: Crear rutas de autenticación

8.1. Editar routes/api.php

Abre routes/api.php y añade las rutas de autenticación:

<?php

use Illuminate\Http\Request;
use Illuminate\Support\Facades\Route;
use App\Http\Controllers\Auth\AuthController;

// Rutas públicas (sin autenticación)
Route::post('/register', [AuthController::class, 'register']);
Route::post('/login', [AuthController::class, 'login']);

// Rutas protegidas (requieren autenticación)
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/user', function (Request $request) {
        return $request->user();
    });
    
    Route::post('/logout', [AuthController::class, 'logout']);
});

// Rutas solo para administradores
Route::middleware(['auth:sanctum', 'admin'])->group(function () {
    Route::get('/admin/dashboard', function () {
        return response()->json(['message' => 'Panel de administrador']);
    });
});

Paso 9: Crear el controlador de autenticación

9.1. Crear el controlador

./vendor/bin/sail artisan make:controller Auth/AuthController

9.2. Editar el controlador

Abre app/Http/Controllers/Auth/AuthController.php y añade:

<?php

namespace App\Http\Controllers\Auth;

use App\Http\Controllers\Controller;
use App\Models\User;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Hash;
use Illuminate\Validation\ValidationException;

class AuthController extends Controller
{
    /**
     * Registrar un nuevo usuario
     */
    public function register(Request $request)
    {
        $request->validate([
            'nombre' => 'required|string|max:255',
            'apellido' => 'required|string|max:255',
            'email' => 'required|string|email|max:255|unique:users',
            'password' => 'required|string|min:8|confirmed',
        ]);

        $user = User::create([
            'nombre' => $request->nombre,
            'apellido' => $request->apellido,
            'email' => $request->email,
            'password' => Hash::make($request->password),
            'is_admin' => false,
        ]);

        $token = $user->createToken('auth-token')->plainTextToken;

        return response()->json([
            'user' => $user,
            'token' => $token,
            'token_type' => 'Bearer',
        ], 201);
    }

    /**
     * Iniciar sesión
     */
    public function login(Request $request)
    {
        $request->validate([
            'email' => 'required|email',
            'password' => 'required',
        ]);

        $user = User::where('email', $request->email)->first();

        if (!$user || !Hash::check($request->password, $user->password)) {
            throw ValidationException::withMessages([
                'email' => ['Las credenciales son incorrectas.'],
            ]);
        }

        // Eliminar tokens anteriores (opcional)
        $user->tokens()->delete();

        $token = $user->createToken('auth-token')->plainTextToken;

        return response()->json([
            'user' => $user,
            'token' => $token,
            'token_type' => 'Bearer',
        ]);
    }

    /**
     * Cerrar sesión
     */
    public function logout(Request $request)
    {
        $request->user()->currentAccessToken()->delete();

        return response()->json([
            'message' => 'Sesión cerrada correctamente',
        ]);
    }
}

Paso 10: Crear middleware de administrador

10.1. Crear el middleware

./vendor/bin/sail artisan make:middleware IsAdmin

10.2. Editar el middleware

Abre app/Http/Middleware/IsAdmin.php y añade:

<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;
use Symfony\Component\HttpFoundation\Response;

class IsAdmin
{
    /**
     * Handle an incoming request.
     */
    public function handle(Request $request, Closure $next): Response
    {
        if (!$request->user() || !$request->user()->isAdmin()) {
            return response()->json([
                'message' => 'No tienes permisos de administrador'
            ], 403);
        }

        return $next($request);
    }
}

10.3. Registrar el middleware

Edita bootstrap/app.php y añade el alias del middleware:

->withMiddleware(function (Middleware $middleware) {
    $middleware->api(prepend: [
        \Laravel\Sanctum\Http\Middleware\EnsureFrontendRequestsAreStateful::class,
    ]);

    $middleware->alias([
        'verified' => \App\Http\Middleware\EnsureEmailIsVerified::class,
        'admin' => \App\Http\Middleware\IsAdmin::class, // Añadir esta línea

})

12.1. Expiración de tokens

Edita config/sanctum.php:

'expiration' => 60 * 24, // Tokens expiran en 24 horas (en minutos)

'expiration' => null,

12.2. Habilidades de tokens (Scopes)

Puedes crear tokens con permisos específicos:

// Crear token con habilidades específicas

// Verificar habilidades en rutas
Route::middleware(['auth:sanctum', 'ability:server:update'])->group(function () {
    // Rutas que requieren la habilidad 'server:update'
});

// Crear diferentes tokens para diferentes dispositivos

Flujo de autenticación

1. Usuario se registra → POST /api/register
   ↓
2. Recibe token de acceso
   ↓
3. Guarda token en localStorage/cookies
   ↓
4. Hace peticiones con header: Authorization: Bearer {token}
   ↓
5. Laravel valida el token con Sanctum
   ↓
6. Si es válido, permite acceso
   ↓
7. Usuario cierra sesión → POST /api/logout
   ↓
8. Token se elimina de la base de datos

Seguridad

Buenas prácticas

  1. Nunca expongas tokens en URLs - Usa headers
  2. Usa HTTPS en producción - Los tokens deben viajar encriptados
  3. Configura expiración de tokens - No uses tokens eternos
  4. Valida todos los inputs - Usa validación de Laravel
  5. Limita intentos de login - Usa throttling
  6. Elimina tokens al cerrar sesión - No dejes tokens activos
  7. Usa SameSite cookies - Protección contra CSRF

Throttling (Límite de peticiones)

Edita routes/api.php:

Route::middleware('throttle:5,1')->group(function () {
    Route::post('/login', [AuthController::class, 'login']);
    Route::post('/register', [AuthController::class, 'register']);
});

Esto limita a 5 intentos por minuto.

Solución de problemas

Error: “Unauthenticated”

Causa: Token inválido o no enviado.

Solución:

  • Verifica que el header sea: Authorization: Bearer {token}
  • Verifica que el token exista en la base de datos
  • Verifica que no haya espacios extra en el token

Error: “CSRF token mismatch”

Causa: Problema con cookies de sesión.

Solución:

./vendor/bin/sail artisan config:clear
./vendor/bin/sail artisan cache:clear

Los tokens no se guardan

Causa: Migración no ejecutada.

Solución:

./vendor/bin/sail artisan migrate

Error 419 en peticiones

Causa: CSRF protection activo en rutas API.

Solución: Las rutas en routes/api.php no deberían tener CSRF. Verifica que uses /api/ en tus URLs.
