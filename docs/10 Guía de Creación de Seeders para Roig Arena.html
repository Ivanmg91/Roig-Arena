Guía de Creación de Seeders para Roig Arena

Esta guía explica cómo crear seeders para poblar la base de datos con datos iniciales y de prueba.

¿Qué son los Seeders?

Los seeders son clases que insertan datos en la base de datos. Son útiles para:

  • Datos iniciales (sectores, asientos)
  • Datos de prueba (usuarios, eventos)
  • Resetear la base de datos rápidamente

Seeders del proyecto

Vamos a crear 5 seeders en este orden:

  1. SectorSeeder - Sectores del estadio (101-122, 301-323, palcos, etc.)
  2. AsientoSeeder - Asientos por sector
  3. UserSeeder - Usuarios de prueba (admin + usuarios normales)
  4. EventoSeeder - Eventos de prueba
  5. PrecioSeeder - Precios por evento/sector

Seeder 1: SectorSeeder

Crear el seeder

sail artisan make:seeder SectorSeeder

Código completo

Ubicación: database/seeders/SectorSeeder.php

<?php

namespace Database\Seeders;

use App\Models\Sector;
use Illuminate\Database\Seeder;

class SectorSeeder extends Seeder
{
    public function run(): void
    {
        $sectores = [];

        // Sectores 101-122
        for ($i = 101; $i <= 122; $i++) {
            $sectores[] = ['nombre' => "Sector $i", 'descripcion' => 'Grada lateral', 'activo' => true];
        }

        // Sectores 301-323
        for ($i = 301; $i <= 323; $i++) {
            $sectores[] = ['nombre' => "Sector $i", 'descripcion' => 'Grada superior', 'activo' => true];
        }

        // Palcos 1-22
        for ($i = 1; $i <= 22; $i++) {
            $sectores[] = ['nombre' => "Palco $i", 'descripcion' => 'Palco VIP', 'activo' => true];
        }

        // Sectores especiales
        $sectores[] = ['nombre' => 'CLUB', 'descripcion' => 'Zona Club', 'activo' => true];
        $sectores[] = ['nombre' => 'JOHNNIE WALKER', 'descripcion' => 'Zona Johnnie Walker', 'activo' => true];
        $sectores[] = ['nombre' => 'PISTA', 'descripcion' => 'Pista central', 'activo' => true];
        $sectores[] = ['nombre' => 'FRONT STAGE', 'descripcion' => 'Frente al escenario', 'activo' => true];

        foreach ($sectores as $sector) {
            Sector::create($sector);
        }

        $this->command->info('✅ Sectores creados: ' . count($sectores));
    }
}

Seeder 2: AsientoSeeder

Crear el seeder

sail artisan make:seeder AsientoSeeder

Código completo

Ubicación: database/seeders/AsientoSeeder.php

<?php

namespace Database\Seeders;

use App\Models\Sector;
use App\Models\Asiento;
use Illuminate\Database\Seeder;

class AsientoSeeder extends Seeder
{
    public function run(): void
    {
        $sectores = Sector::all();
        $totalAsientos = 0;

        foreach ($sectores as $sector) {
            $asientos = $this->generarAsientosPorSector($sector);
            $totalAsientos += count($asientos);
        }

        $this->command->info("✅ Asientos creados: {$totalAsientos}");
    }

    private function generarAsientosPorSector(Sector $sector): array
    {
        $asientos = [];

        // Sectores 101-122 y 301-323: 20 filas x 15 asientos = 300 asientos
        if (preg_match('/^Sector (10[1-9]|1[1-2][0-9]|30[1-9]|3[1-2][0-9])$/', $sector->nombre)) {
            for ($fila = 1; $fila <= 20; $fila++) {
                for ($numero = 1; $numero <= 15; $numero++) {
                    $asientos[] = Asiento::create([
                        'sector_id' => $sector->id,
                        'fila' => (string) $fila,
                        'numero' => $numero,
                    ]);
                }
            }
        }
        // Palcos: 1 fila x 8 asientos = 8 asientos
        elseif (str_starts_with($sector->nombre, 'Palco')) {
            for ($numero = 1; $numero <= 8; $numero++) {
                $asientos[] = Asiento::create([
                    'sector_id' => $sector->id,
                    'fila' => 'A',
                    'numero' => $numero,
                ]);
            }
        }
        // CLUB: 10 filas x 20 asientos = 200 asientos
        elseif ($sector->nombre === 'CLUB') {
            for ($fila = 1; $fila <= 10; $fila++) {
                for ($numero = 1; $numero <= 20; $numero++) {
                    $asientos[] = Asiento::create([
                        'sector_id' => $sector->id,
                        'fila' => (string) $fila,
                        'numero' => $numero,
                    ]);
                }
            }
        }
        // JOHNNIE WALKER: 8 filas x 15 asientos = 120 asientos
        elseif ($sector->nombre === 'JOHNNIE WALKER') {
            for ($fila = 1; $fila <= 8; $fila++) {
                for ($numero = 1; $numero <= 15; $numero++) {
                    $asientos[] = Asiento::create([
                        'sector_id' => $sector->id,
                        'fila' => (string) $fila,
                        'numero' => $numero,
                    ]);
                }
            }
        }
        // PISTA: 30 filas x 25 asientos = 750 asientos
        elseif ($sector->nombre === 'PISTA') {
            for ($fila = 1; $fila <= 30; $fila++) {
                for ($numero = 1; $numero <= 25; $numero++) {
                    $asientos[] = Asiento::create([
                        'sector_id' => $sector->id,
                        'fila' => (string) $fila,
                        'numero' => $numero,
                    ]);
                }
            }
        }
        // FRONT STAGE: 5 filas x 30 asientos = 150 asientos
        elseif ($sector->nombre === 'FRONT STAGE') {
            for ($fila = 1; $fila <= 5; $fila++) {
                for ($numero = 1; $numero <= 30; $numero++) {
                    $asientos[] = Asiento::create([
                        'sector_id' => $sector->id,
                        'fila' => (string) $fila,
                        'numero' => $numero,
                    ]);
                }
            }
        }

        return $asientos;
    }
}

Seeder 5: PrecioSeeder

Crear el seeder

sail artisan make:seeder PrecioSeeder

Código completo

Ubicación: database/seeders/PrecioSeeder.php

<?php

namespace Database\Seeders;

use App\Models\Evento;
use App\Models\Sector;
use App\Models\Precio;
use Illuminate\Database\Seeder;

class PrecioSeeder extends Seeder
{
    public function run(): void
    {
        $eventos = Evento::all();
        $sectores = Sector::all();
        $totalPrecios = 0;

        foreach ($eventos as $evento) {
            foreach ($sectores as $sector) {
                $precio = $this->calcularPrecio($evento, $sector);
                
                Precio::create([
                    'evento_id' => $evento->id,
                    'sector_id' => $sector->id,
                    'precio' => $precio,
                    'disponible' => true,
                ]);
                
                $totalPrecios++;
            }
        }

        $this->command->info("✅ Precios creados: {$totalPrecios}");
    }

    private function calcularPrecio(Evento $evento, Sector $sector): float
    {
        // Precios base según tipo de sector
        $precioBase = match(true) {
            str_starts_with($sector->nombre, 'Palco') => 150.00,
            $sector->nombre === 'FRONT STAGE' => 120.00,
            $sector->nombre === 'CLUB' => 100.00,
            $sector->nombre === 'JOHNNIE WALKER' => 90.00,
            $sector->nombre === 'PISTA' => 80.00,
            str_starts_with($sector->nombre, 'Sector 10') => 50.00, // 101-122
            str_starts_with($sector->nombre, 'Sector 30') => 40.00, // 301-323
            default => 50.00,
        };

        // Multiplicador según tipo de evento
        $multiplicador = match($evento->nombre) {
            'Final Copa del Rey' => 1.5,
            'Concierto Rock 2026' => 1.3,
            'Festival Electrónica' => 1.2,
            default => 1.0,
        };

        return round($precioBase * $multiplicador, 2);
    }
}

Lógica de precios:

  • Palcos: 150€ base
  • Front Stage: 120€ base
  • Club: 100€ base
  • Johnnie Walker: 90€ base
  • Pista: 80€ base
  • Sectores 101-122: 50€ base
  • Sectores 301-323: 40€ base

Multiplicadores por evento:

  • Final Copa del Rey: ×1.5
  • Concierto Rock: ×1.3
  • Festival Electrónica: ×1.2
  • Teatro: ×1.0

Apéndice: Seeders en Laravel para Principiantes

Esta guía te ayudará a entender qué son los seeders, cómo funcionan y cómo usarlos correctamente.

¿Qué son los Seeders?

Los seeders son clases PHP que insertan datos en la base de datos. Son como “plantadores de semillas” que pueblan tu base de datos con información inicial o de prueba.

Analogía simple

Imagina que construyes una casa:

  • Migraciones = Construir las habitaciones (estructura)
  • Seeders = Amueblar las habitaciones (datos)

Las migraciones crean las tablas vacías, los seeders las llenan con datos.

¿Por qué usar Seeders?

Problema sin Seeders

-- Insertar datos manualmente cada vez
INSERT INTO users (nombre, email, password) VALUES ('Juan', 'juan@example.com', '...');
INSERT INTO users (nombre, email, password) VALUES ('María', 'maria@example.com', '...');
-- Repetir en cada entorno (local, staging, producción)

Problemas:

  • ❌ Tedioso y propenso a errores
  • ❌ Difícil de compartir con el equipo
  • ❌ No versionado en Git
  • ❌ Inconsistente entre entornos

Solución con Seeders

// database/seeders/UserSeeder.php
User::create(['nombre' => 'Juan', 'email' => 'juan@example.com', ...]);
User::create(['nombre' => 'María', 'email' => 'maria@example.com', ...]);

Ventajas:

  • ✅ Automatizado
  • ✅ Versionado en Git
  • ✅ Compartible con el equipo
  • ✅ Consistente en todos los entornos
  • ✅ Fácil de resetear

Estructura de un Seeder

Ubicación

database/seeders/
├── DatabaseSeeder.php    # Seeder principal (orquestador)
├── UserSeeder.php
├── PostSeeder.php
└── CategorySeeder.php

Anatomía básica

<?php

// 1. NAMESPACE
namespace Database\Seeders;

// 2. IMPORTS
use App\Models\User;
use Illuminate\Database\Seeder;
use Illuminate\Support\Facades\Hash;

// 3. CLASE - Extiende Seeder
class UserSeeder extends Seeder
{
    /**
     * 4. MÉTODO RUN - Ejecuta el seeder
     */
    public function run(): void
    {
        // Insertar datos aquí
        User::create([
            'nombre' => 'Juan',
            'email' => 'juan@example.com',
            'password' => Hash::make('password'),
        ]);
    }
}

Imports necesarios

1. Seeder (obligatorio)

use Illuminate\Database\Seeder;

¿Qué es? Clase base de todos los seeders.
¿Para qué? Proporciona la estructura básica.

2. Modelos

use App\Models\User;
use App\Models\Post;

¿Qué son? Los modelos Eloquent que vas a usar.
¿Para qué? Insertar datos usando Eloquent.

3. Facades útiles

use Illuminate\Support\Facades\Hash;  // Para hashear passwords
use Illuminate\Support\Facades\DB;    // Para queries directas

Método run()

El método run() es donde defines qué datos insertar.

public function run(): void
{
    // Tu código aquí
}

Características:

  • Se ejecuta cuando llamas al seeder
  • Puede insertar uno o muchos registros
  • Puede llamar a otros seeders

Formas de insertar datos

1. Eloquent create() - Recomendado

User::create([
    'nombre' => 'Juan',
    'email' => 'juan@example.com',
    'password' => Hash::make('password'),
]);

Ventajas:

  • ✅ Usa el modelo (validaciones, eventos)
  • ✅ Actualiza timestamps automáticamente
  • ✅ Más legible

2. Eloquent insert() - Múltiples registros

User::insert([
    ['nombre' => 'Juan', 'email' => 'juan@example.com', 'created_at' => now()],
    ['nombre' => 'María', 'email' => 'maria@example.com', 'created_at' => now()],
]);

Ventajas:

  • ✅ Más rápido para muchos registros
  • ❌ No dispara eventos del modelo
  • ❌ Debes añadir timestamps manualmente

3. DB::table() - Query directa

DB::table('users')->insert([
    'nombre' => 'Juan',
    'email' => 'juan@example.com',
    'created_at' => now(),
]);

Cuándo usar:

  • Cuando no tienes modelo
  • Para tablas pivot
  • Para máximo rendimiento

4. Factories - Datos aleatorios

User::factory()->count(10)->create();

Cuándo usar:

  • Para generar muchos datos de prueba
  • Cuando no importan los valores exactos

DatabaseSeeder (Orquestador)

El archivo DatabaseSeeder.php es el seeder principal que llama a los demás.

<?php

namespace Database\Seeders;

use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    public function run(): void
    {
        // Llamar a otros seeders en orden
        $this->call([
            UserSeeder::class,
            CategorySeeder::class,
            PostSeeder::class,
        ]);
    }
}

Orden importa:

  • Primero las tablas sin foreign keys
  • Luego las que dependen de otras

Ejemplo:

$this->call([
    UserSeeder::class,      // 1. Usuarios (sin dependencias)
    CategorySeeder::class,  // 2. Categorías (sin dependencias)
    PostSeeder::class,      // 3. Posts (depende de users y categories)
]);

Patrones comunes

Patrón 1: Datos fijos

public function run(): void
{
    $roles = ['admin', 'editor', 'viewer'];
    
    foreach ($roles as $role) {
        Role::create(['nombre' => $role]);
    }
}

Patrón 2: Datos con relaciones

public function run(): void
{
    $user = User::create([
        'nombre' => 'Juan',
        'email' => 'juan@example.com',
    ]);
    
    // Crear posts para este usuario
    Post::create([
        'user_id' => $user->id,
        'titulo' => 'Mi primer post',
    ]);
}

Patrón 3: Verificar antes de insertar

public function run(): void
{
    if (User::where('email', 'admin@example.com')->exists()) {
        $this->command->info('Admin ya existe, saltando...');
        return;
    }
    
    User::create([
        'email' => 'admin@example.com',
        // ...
    ]);
}

Patrón 4: Datos desde array

public function run(): void
{
    $users = [
        ['nombre' => 'Juan', 'email' => 'juan@example.com'],
        ['nombre' => 'María', 'email' => 'maria@example.com'],
        ['nombre' => 'Carlos', 'email' => 'carlos@example.com'],
    ];
    
    foreach ($users as $userData) {
        User::create($userData);
    }
}

Patrón 5: Datos desde archivo

public function run(): void
{
    $json = file_get_contents(database_path('data/users.json'));
    $users = json_decode($json, true);
    
    foreach ($users as $user) {
        User::create($user);
    }
}

Patrón 6: Usar factories

public function run(): void
{
    // Crear 50 usuarios aleatorios
    User::factory()->count(50)->create();
    
    // Crear 10 usuarios con estado específico
    User::factory()->count(10)->create([
        'is_admin' => true,
    ]);
}

Hashear passwords

Siempre hashea las contraseñas:

use Illuminate\Support\Facades\Hash;

User::create([
    'nombre' => 'Juan',
    'email' => 'juan@example.com',
    'password' => Hash::make('password'),  // ← Hashear
]);

❌ Nunca hagas esto:

'password' => 'password',  // ❌ Texto plano

Mensajes informativos

Usa $this->command para mostrar mensajes:

public function run(): void
{
    User::create([...]);
    
    $this->command->info('✅ Usuario creado correctamente');
    $this->command->warn('⚠️ Advertencia: datos de prueba');
    $this->command->error('❌ Error al crear usuario');
}

Por existencia de datos

public function run(): void
{
    if (User::count() > 0) {
        $this->command->info('Ya existen usuarios, saltando...');
        return;
    }
    
    // Insertar solo si la tabla está vacía
    User::create([...]);
}

Truncar tablas antes de insertar

use Illuminate\Support\Facades\DB;

public function run(): void
{
    // Desactivar foreign keys temporalmente
    DB::statement('SET FOREIGN_KEY_CHECKS=0;');
    
    // Vaciar tabla
    User::truncate();
    
    // Reactivar foreign keys
    DB::statement('SET FOREIGN_KEY_CHECKS=1;');
    
    // Insertar datos
    User::create([...]);
}

Seeders vs Factories

Seeders

  • Datos específicos y controlados
  • Datos de producción (roles, configuración)
  • Pocos registros
// Seeder - Datos específicos
User::create([
    'nombre' => 'Admin',
    'email' => 'admin@example.com',
    'is_admin' => true,
]);

Factories

  • Datos aleatorios
  • Datos de prueba
  • Muchos registros
// Factory - Datos aleatorios
User::factory()->count(100)->create();

Errores comunes

Error: “Class not found”

Causa: Falta el import del modelo.
Solución:

use App\Models\User;

Error: “SQLSTATE[23000]: Integrity constraint violation”

Causa: Violación de foreign key o unique.
Solución:

  • Verificar orden de seeders
  • Verificar que existan registros relacionados
  • Verificar duplicados

Error: “Mass assignment exception”

Causa: Campo no está en $fillable.
Solución:

// En el modelo
protected $fillable = ['nombre', 'email', 'password'];

Error: Seeders no se ejecutan

Causa: No están registrados en DatabaseSeeder.
Solución:

$this->call([
    UserSeeder::class,  // ← Añadir aquí
]);
